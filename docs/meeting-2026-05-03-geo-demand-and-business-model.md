# 会议纪要 2026-05-03 —— GEO 需求侧 + 商业模式讨论

## 0. 元信息 / 上下文

本次会话起点：用户拿到 GEO 服务商「探词科技」的产品手册 PDF（21 页 PowerPoint），希望把它转 markdown 后融入仓库研究。完成话术拆解卡之后，讨论自然延伸到三个**仓库当前结构性缺失**的根本问题：

1. 探词的真实技术栈是什么样的？
2. 它的商业模式合理性在哪里？
3. **GEO 究竟有没有真实需求基础？** 什么用户、什么问题真的用 LLM 找答案？

第三个问题尤其关键——它**前置于仓库现有 14 张技术卡的所有讨论**：信号优化的有效性是次要问题，**先得有"用户在用 LLM 做这类决策"为前提**。

这是仓库自启动以来第一次系统讨论需求侧。

## 1. 探词业务的技术反向工程

### 1.1 一句定性

**80% 是新闻稿矩阵的 LLM 时代版本，20% 是 LLM 监测仪表盘**。

新瓶装旧酒：
- 新瓶 = 监测大屏（11 平台 × 10 万+ 提问模板）
- 旧酒 = 7 万+ 媒体发布渠道（采购自传统软文 / 新闻稿分发产业链）

这与手册第 7 页里它极力撇清的"传统 GEO / 新闻稿供应商"在结构上是**同一类公司**。

### 1.2 四模块反向工程

| 手册模块 | 推测的真实实现 | 难度 | 与仓库的关系 |
|---------|--------------|------|------------|
| 01 现状分析 | 11 LLM × 10w+ query 爬虫监测 + NER + 关键词匹配检测品牌提及/位次/引用源 | 中（卡在 selector 稳定性 + 账号管理 + 反爬） | [`tools/exp-doubao-vs-deepseek-paired/`](../tools/exp-doubao-vs-deepseek-paired/) 的工业化扩展版 |
| 02 深度分析 | 数据切片 + 文字模板填充的报告系统 | 低 | 客户感知"专业度"主要来自这层，但技术含量最低 |
| 03 内容生成 | 大模型按"弱点 query"生成针对性图文 | 低-中 | 反向用 [`prompt-level-seo-cn.md`](../techniques/prompt-level-seo-cn.md) 逻辑：让 LLM 生产内容去 hack 另一个 LLM 检索 |
| 04 内容发布 | 对接 7 万+ 新闻稿 / 软文分发渠道 | 低（采购整合） | **真正决定效果的层** |

### 1.3 核心技术判断

**真正起作用的不是前三个模块，是第四个模块**。

理由：仓库三条强假说同时指向"AI 适配"动作的边际效应近 0：
- [`structured-data-cn-practice.md`](../techniques/structured-data-cn-practice.md) 元假说 B（schema）
- [`llms-txt-cn-practice.md`](../techniques/llms-txt-cn-practice.md) 元假说 A（llms.txt）
- [`multimodal-cn.md`](../techniques/multimodal-cn.md) 元假说 B（alt）

所以单条信号没用，**只能靠"在 7 万家收录站上铺量"形成信源池占有率**，让 LLM 在 RAG / 训练 / 实时检索三条路径里至少命中一条。

[`private-vs-public-source-cn.md`](../techniques/private-vs-public-source-cn.md) 元假说 B 已指出公众号矩阵仅元宝 / 微信生效，但**新闻稿矩阵在百度收录的网站上对豆包 / 文心 / 千问的实时检索路径仍然有效**——这正是探词的核心生意空间。

## 2. 探词的商业模式反推

### 2.1 价格表反推真实业务结构

```
SaaS  9999 元/年        ← 漏斗顶端引流款
RaaS  3000/季 + 5000/季 ← 主力利润款（一个 query 9900/年）
OEM   58888 元（源码）   ← 卖给同行做白标
```

**RaaS 一个关键词的年费 ≈ SaaS 整套工具年费**——SaaS 几乎不指望直接赚钱，是引流 / 蹭名声。真正赚钱的是 RaaS 代运营。

客户列表（奥迪、江铃、安邦制药、宁夏圆润）大头都在 RaaS——说明手册里"算法优势 / 90% 适配"的故事讲完，落到交付时还是**人工服务 + 软文铺量**。

OEM 58888 卖源码 = **给同行卖工具**，对应 SEO 行业"千博 / 高科 / 蝙蝠侠"白标系统提供商的成熟玩法，换 GEO 标签接着做。

### 2.2 渠道战略暴露交付不可标准化

p20 写得很坦率：湖南 3 家、深圳 1 家、上海 1 家代理。SaaS 类标准化产品根本不需要这么重的代理网络。代理商的存在反向证明：

- 客户教育成本高（GEO 是新词，要面对面讲）
- 交付强依赖本地关系（代理商懂客户行业 + 媒体关系）
- 单价高（不然代理商赚不到钱）

跟 [`geo-roi-framework-cn.md`](../techniques/geo-roi-framework-cn.md) 元假说 A "中小新品牌中文 GEO ROI 结构性低于英文" 是同一现象的供需两面：买方 ROI 不稳，卖方就只能靠人工 + 关系堆，无法纯软件交付。

### 2.3 客户画像分层 = 三种产品分别打三种钱

| 画像 | 对应产品 | 卖什么 |
|------|--------|--------|
| 大型 B2B（奥迪、安邦制药等） | RaaS | 卖**人工 + 媒体关系**，"AI 算法"是包装 |
| B2C 电商 / 大型企业市场部 | SaaS | 卖**仪表盘焦虑** —— "你看你在 AI 里隐身了" |
| 营销服务商（同行） | OEM / 源码 | 卖**白标产品** —— 让同行去找他们自己的客户 |

OEM 这一层把探词从"服务商"变成"军火商"——同行越多家用探词的 OEM，"GEO 是趋势"的市场叙事就越自动放大，反过来抬高 RaaS 主营业务的可信度。这是中国营销 SaaS 行业的成熟玩法。

## 3. GEO 需求侧 —— 什么用户 × 什么问题真的用 LLM

> **本节是仓库当前结构性最大缺失**，决定下一张需求侧卡的内容。

### 3.1 用户类型轴

| 用户层 | 用 LLM 找答案概率 | 占总人口 | 备注 |
|-------|----------------|---------|------|
| 极客 / IT 从业者 | 高 | < 5% | 重度使用，但不是 GEO 商业目标 |
| 白领 / 知识工作者 | 中-高 | ~20% | 多数"工作辅助"，**消费决策仍走小红书 / 淘宝** |
| 学生（中学到研究生） | 高 | ~10% | 真"找答案"，但**消费力低** |
| 新中产消费者 | 低-中 | ~15% | LLM 是补充而非首选 |
| 银发 / 下沉市场 | 极低 | ~50% | 微信 / 抖音 / 百度，根本不进 LLM 入口 |

### 3.2 问题类型轴

| 问题类型 | LLM 渗透度 | GEO 商业价值 |
|---------|-----------|------------|
| 高频低决策（翻译/算账/写邮件） | 极高 | **零** —— 没人推广翻译里的品牌 |
| 学习 / 研究类 | 高 | **ToB 学术工具有价值** |
| **B2B 选型** | **高** | **⭐ 真正有效** |
| **YMYL 初筛** | 中-高 | **⭐ 部分有效** |
| **新概念 / 长尾决策** | 中-高 | **⭐ LLM 独占场景** |
| **跨语言 / 跨文化** | 高 | **⭐ 出海双向缺供给** |
| 高客单消费（买房/车/保险） | 中 | **低** |
| 本地服务（餐厅/医院/律师） | **低** | **极低**（LLM 没 LBS） |
| 冲动消费（美妆/零食/衣服） | **极低** | **零**（全在小红书 / 抖音） |

### 3.3 关键结论

**GEO 真正有商业空间的场景只有 4 类**：B2B 选型、YMYL 初筛、新概念 / 长尾决策、跨语言陪跑。

回看探词手册 6 类客户画像——B2C 电商 / 跨境出海 / 品牌公关 / 中小微企业主 这 **4 类几乎全部踩在 GEO 无效区**。客户画像越宽 = 销售在勉强匹配市场。

**真正能稳定吃到 GEO 红利的只有 2 类**：B2B 市场增长总监 + 营销服务商自用。

## 5. 探词合理性 / 缺陷 / 结构性风险

### 5.1 合理性

1. 四产品线矩阵理论上覆盖全谱客户，且 OEM 让同行扩大叙事——叙事自动放大型生意
2. **效果费按"单关键问题"切分**——把不可承诺的"业务效果"降维成可验证的"单 query 位次"，是聪明的产品化
3. 代理商 + 标杆直营符合中国 B2B 营销服务市场规律
4. 湖南本地起家避开北上深红海
5. 客户列表里真正成功的部分（百奥云 / 世邦通信 / 安邦制药）确实落在 GEO 有效区

### 5.2 缺陷

1. **重交付陷阱**：RaaS 主力但每单需人工，团队结构（3+3+3+1）已是咨询公司配比
2. **6 类客户画像里 4 类不是真买家**——销售物料在勉强匹配市场
3. **11 平台监测的技术债指数级增长**——任何平台改 UI 都要返工
4. **OEM 卖源码长期自伤**——同行二开做出更便宜版本（千博系统剧本）
5. **缺独特数据资产**——21631 提及全是公开抓的，无壁垒
6. **价格锚点卡中间**——9900/年对大客户太小、中小客户太大
7. **效果不可独立验证**——客户没法做反事实

### 5.3 结构性风险（影响整行业）

1. **信源铺量靠采购**，议价权在上游媒体
2. **LLM 平台一旦自建广告位**（豆包 / 元宝 / 千问任意一家做 LLM 时代竞价排名），GEO 中介价值瞬间腰斩——悬在所有 GEO 公司头上的达摩克利斯之剑
3. **C-SEO Bench 类研究若进一步公开化**，市场叙事根基松动——仓库存在意义之一就在这

## 6. 值得探索的方向（按优先级）

| # | 方向 | 为什么值得做 |
|---|------|------------|
| 1 | **GEO 战略咨询** | 避开效果承诺、轻交付、按项目计费、毛利高；正适合仓库 14 张卡产品化 |

| 3 | **YMYL 合规 GEO 垂直陪跑** | 公关公司不懂技术、技术公司不懂合规——夹缝清晰 |
| 4 | **反 GEO 监测** | 进攻型行业必长出防御型，付费意愿不亚于进攻方 |

| 6 | **企业内 GEO（私有 RAG / 企业 AI 助手内容优化）** | 多数 GEO 公司忽略私有部署，**这才是更大盘子** |
| 7 | **协议 / 标准推动** | 第一个推规则的人有定价权 |
