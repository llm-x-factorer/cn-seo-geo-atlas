# 6 平台 × 4 形态平台变现监测系统

> ⚠️ **实验状态**：**设计完成 + T0 基线建立**。本文档建立监测协议 + 当前快照（2026-05-04）。后续每季度全面更新 + 任何平台变现形态上线即告警。
>
> **价值定位**：仓库 DS 元假说 A 升级版"4 形态平台变现冲击"的实证基础设施——把审计后被升级的反话术假说转为**可纵向追踪的监测告警系统**。低成本（自建爬虫 + 公开文档追踪 + 媒体报道追踪）/ 高产出（任一形态上线即引发仓库 DS 维度全部假说重审）。

## 一、假设

### 主假设（H1）

> 4 形态中，**形态 1 + 形态 2 + 形态 4 已在 2025 年部分上线**，**形态 3（自建品牌方监测后台）尚未在中文场景上线**。

### 子假设

- **H1.1**：形态 1（订阅制）—— 至少 3/6 平台已上线消费侧付费版（豆包 / 文心一言 / Kimi）
- **H1.2**：形态 2（电商佣金）—— 至少 2/6 平台已链接电商通路（豆包 → 抖音商店 / 千问 → 淘宝）
- **H1.3**：形态 3（自建品牌方监测后台）—— 0/6 平台上线，预计 12-24 个月内某平台上线
- **H1.4**：形态 4（API 收费）—— 4/6 平台已商业化（豆包 / 千问 / 元宝 / 文心）

### 长期假设（H2）

> 任一形态在某平台上线后 **6 个月内**，仓库 [`techniques/geo-business-model-taxonomy-cn.md`](../techniques/geo-business-model-taxonomy-cn.md) c/d 层（内容铺量 + 效果代运营）中介客户的实际付费意愿 / 续约率出现可观测下降。

## 二、4 形态监测协议

### 形态 1：订阅制（消费侧绕过）

**机制**：消费侧用户付费给平台，平台从用户口袋直接收钱。

**监测信号**：

| 信号 | 监测方法 | 告警阈值 |
|---|---|---|
| 平台上线消费侧订阅版 | 平台官网 + 应用市场抓取定价页 | 任一平台首次上线 = 告警 |
| 订阅版用户数披露 | 投资者关系材料 + 季报 + 媒体报道 | ≥ 100 万订阅用户 = 告警（标志规模化） |
| 订阅版与免费版答案质量分层 | 实测对照（同 query 在订阅 / 免费版的引用源池差异） | 答案质量差异 ≥ 30% = 告警（标志付费用户对训练 / RAG 数据源筛选有结构性影响） |

**当前 T0 基线（2026-05-04）**：

| 平台 | 订阅版状态 | 公开定价 | T0 数据源 |
|---|---|---|---|
| **豆包** | ✅ 已上线 | 付费版（按月 / 按年套餐，具体定价待查应用市场） | 火山方舟豆包助手计费文档 + 智推创始人原话 |
| **文心一言** | ✅ 已上线 | 会员 49.9-59.9 元/月（2025）| 智推创始人原话 + 媒体报道 |
| **Kimi** | ✅ 已上线 | Kimi 探索版按 query 计费（先锋玩家）| 智推创始人原话 |
| **元宝** | ⚠️ 部分（未独立订阅版，绑定微信生态）| n/a | 公开物料 |
| **千问** | ⚠️ 部分（绑定阿里生态）| n/a | 公开物料 |
| **小红书问一问** | ❌ 未上线 | n/a | 平台未公开商业化 |

**结论**：H1.1 满足——3/6 平台（豆包 / 文心 / Kimi）已上线消费侧订阅版。

### 形态 2：电商佣金（供给侧绕过）

**机制**：平台直接抽供给侧成交佣金，把答案体直接连到电商通路，跳过 c/d 层的内容铺量。

**监测信号**：

| 信号 | 监测方法 | 告警阈值 |
|---|---|---|
| 答案体出现商品卡 | 实测同 query 在 6 平台答案体中商品卡 / 跳转店铺出现频率 | 任一平台首次出现 = 告警 |
| 平台抽佣率公开数据 | 平台商家中心文档 + 招商信息 | 公开披露抽佣率 = 告警 |
| 电商通路与答案体的深度集成 | 实测从 LLM 答案到下单的完整路径 | 端到端 ≤ 2 跳 = 告警（标志深度集成）|

**当前 T0 基线（2026-05-04）**：

| 平台 | 电商通路集成状态 | 抽佣率 | T0 数据源 |
|---|---|---|---|
| **豆包** | ✅ 链接抖音商店 | n/a（待查招商文档）| 智推创始人原话 2025-11-28 |
| **千问** | ✅ 与淘宝深度绑定（"AI 下单"路径）| n/a（淘宝既有抽佣体系）| 智推创始人原话 |
| **元宝** | ⚠️ 可路由微信小程序 + 视频号小店 | n/a | 仓库 [`platforms/yuanbao/`](../platforms/yuanbao/) |
| **文心** | ❌ 未明确电商集成 | n/a | 待查 |
| **Kimi** | ❌ 未明确电商集成 | n/a | 待查 |
| **小红书问一问** | ⚠️ 与小红书电商生态共用 | n/a | 待查 |
| **海外对照（ChatGPT）** | ✅ ChatGPT Atlas 跳蓝链 → Shopify | n/a | 智推创始人原话 |

**结论**：H1.2 满足——至少 2/6 平台（豆包 + 千问）已链接电商通路。

### 形态 3：自建品牌方监测后台

**机制**：LLM 平台上线"品牌方查询自家在 LLM 中表现"的官方后台（类比 Google Search Console for SEO）。

**监测信号**：

| 信号 | 监测方法 | 告警阈值 |
|---|---|---|
| 平台官方"商家品牌可见度查询"模块上线 | 平台开发者中心 / 商家中心 / 品牌中心抓取 | 任一平台首次上线 = 告警（高优先级） |
| 平台 B 端 / 商家中心新增 GEO 相关 dashboard | 平台 B 端后台抓取 | 任一新增 = 告警 |
| 平台官方文档新增"品牌可见度"相关 API | 平台 API 文档抓取 | 任一新增 = 告警 |

**当前 T0 基线（2026-05-04）**：

| 平台 | 品牌方监测后台状态 | T0 数据源 |
|---|---|---|
| **豆包 / 火山引擎** | ❌ 未上线 | 火山方舟文档（已查无相关模块）|
| **文心 / 千帆** | ❌ 未上线 | 待查百度智能云控制台 |
| **千问 / 百炼** | ❌ 未上线 | 待查阿里云百炼平台 |
| **元宝 / 腾讯云** | ❌ 未上线 | 待查腾讯云控制台 |
| **Kimi / Moonshot** | ❌ 未上线 | 待查 Moonshot 开发者平台 |
| **小红书问一问** | ❌ 未上线 | 平台无公开 B 端后台 |

**结论**：H1.3 满足——**0/6 平台已上线**自建品牌方监测后台。这是 4 形态中唯一在中文场景"尚未发生"的形态——也是最高优先级监测点。

⚠️ **历史先例参照**：Google Search Console（2015 升级为现名）历经 9 年迭代成为标准 SEO 工具。中文 LLM 平台对应工具的诞生预计 12-24 个月内开始（基于 SEO → GEO 周期类比缩短）。

### 形态 4：API 收费（B2B 端绕过）

**机制**：平台对 B2B 端开发者直接收 API 费（按 token / 按调用），让企业客户**自己接入 LLM 能力**而不是通过中间服务商。

**监测信号**：

| 信号 | 监测方法 | 告警阈值 |
|---|---|---|
| 平台 API 公开定价 | 平台开发者文档 + 商业化页面 | 任一平台首次公开定价 = 告警 |
| 企业 API 调用量披露 | 平台官方公告 + 投资者关系材料 + 季报 | 调用量同比增长 ≥ 100% = 告警（标志爆发式增长）|
| 企业级 API 套餐 / SLA | 平台企业版商业化页面 | 任一平台推 SLA 套餐 = 告警 |

**当前 T0 基线（2026-05-04）**：

| 平台 | API 商业化状态 | 公开定价（输入 / 输出，元/百万 tokens） | T0 数据源 |
|---|---|---|---|
| **豆包** / 火山引擎 | ✅ 已商业化 | doubao-seed-2.0-pro: **3.2 / 16**；lite: 0.6 / 3.6；mini: 0.2 / 2 | 火山引擎 product/doubao 页 |
| **千问** / 阿里百炼 | ✅ 已商业化 | （待查百炼定价页） | 智推创始人原话 + 阿里 5 年云+AI 1000 亿美金目标 |
| **元宝** / 腾讯云混元 | ✅ 已商业化 | （待查腾讯云）| 智推创始人原话 |
| **文心** / 百度千帆 | ✅ 已商业化 | （待查千帆定价页）| 公开物料 |
| **Kimi** / Moonshot | ⚠️ 部分（开发者平台）| （待查 Moonshot）| 公开物料 |
| **小红书问一问** | ❌ 未上线 | n/a | 平台无公开 API |

**结论**：H1.4 部分满足——**4-5/6 平台**（豆包 / 千问 / 元宝 / 文心 +/- Kimi）已商业化 API。豆包 API 定价已公开，是仓库唯一可获 A 级 API 定价数据。

## 三、4 形态监测节奏

### 季度全面（每 3 个月）

- 6 平台 × 4 形态完整监测一遍
- 输出"季度变化报告"——记录每形态的状态变化（新上线 / 价格调整 / 用户数披露等）
- 同步更新 [`techniques/geo-business-model-taxonomy-cn.md`](../techniques/geo-business-model-taxonomy-cn.md) 元假说 A 升级版的具体形态状态

### 月度告警（每月）

- 仅监测高优先级信号（形态 3 平台上线 / 形态 1 用户数披露 / 形态 4 调用量披露 / 形态 2 商品卡突变）
- 任何告警触发即引发仓库 DS 维度假说集中重审

### 一次性事件（任一时刻）

- 任一平台首次上线品牌方监测后台 → 立即触发**仓库 DS 元假说 A 升级版重写** + [`anti-geo-monitoring-cn.md`](../techniques/anti-geo-monitoring-cn.md) 元假说 G 立即升级
- 任一平台 API 调用量同比增长 ≥ 100% → 触发 c/d 层影响分析
- 任何 GEO 服务商公开提及"被平台变现冲击" → 触发服务商话术演化卡更新

## 六、不做的事

- **不监测平台答案质量主观评分**——这是信号侧 D1-D8 维度的工作，与本实验（供给侧 DS）独立
- **不直接对接平台商业部门 / BD**——本实验完全基于公开物料追踪，不涉及内部信息
- **不预测平台的具体行为时间表**——只监测已发生的事件，不预测未来
- **不用本实验数据评判头部 / 腰部 / 尾部 GEO 公司的存亡**——存亡判断由 [`techniques/geo-industry-stratification-cn.md`](../techniques/geo-industry-stratification-cn.md) 元假说 C 单独执行
- **不用本实验数据反对头部公司的横向扩展路径**——迈富时 / 百分点 h 层路径与本实验测的中介风险不冲突
