# experiments/

可复现实验记录。和 `case-studies/` 的区别：

| | case-studies | experiments |
|---|---|---|
| 设计 | 事后复盘 | 事前假设 + 受控变量 |
| 样本 | 真实业务，单点 | 可重复，最好多样本 |
| 数据 | 业务后台 | 受控收集，可被第三方复跑 |
| 输出 | 经验 / 启示 | 信号关系 / 机制证据 |

## 一份合格的实验记录包含

1. **假设**：明确陈述"如果做 X，会观察到 Y"
2. **方法**：信号、控制变量、样本量、时间窗口、采集工具
3. **数据**：原始数据（脱敏可接受）、统计方式
4. **结果**：支持假设 / 否定假设 / 不确定
5. **可复现性**：别人按你的方法重跑，需要什么环境、数据源、权限

## 实验类型示例

- 在 N 个相似账号上分别用 A / B 两种结构发同主题内容，观察站内搜索曝光
- 同一 query 每天定时在 5 个生成式引擎上跑 30 次，记录引用源分布
- 修改某站结构化数据后，观察百度搜索结果中特型卡的出现率变化
- 把同一份内容拆成不同长度 / 段落结构，观察 LLM 引用率

## 模板

复制 [`../templates/experiment-template.md`](../templates/experiment-template.md)。

## 命名

`<主题>-<日期>.md`，例如 `kimi-citation-source-distribution-2026-04.md`。

## 数据放哪

- 小数据（< 1MB）：直接放在实验记录同目录的 `data/` 子目录（注意：会让目录深度到第三层，仅在数据真的必要时使用，否则放 gist / 外部存储并贴链接）
- 大数据 / 隐私数据：外部存储 + 校验和
