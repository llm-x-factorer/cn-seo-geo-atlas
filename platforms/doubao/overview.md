# 豆包平台概况

> ⚠️ **状态说明**：本卡片基于公开报道与第三方 GEO 服务商二手信息整理，**未经作者亲自实测**。机制层面的论断标注了来源；服务商口径的数据（如"今日头条占引用 60%+"）置于"待验证"。最近一次实测复核计划：见末尾"未解问题"。

## 基本信息

| 字段 | 值 |
|------|----|
| 产品方 | 字节跳动 / 火山引擎 |
| 底层模型 | 豆包大模型（系列） |

## 战略定位

- **独立 APP 路线**：相比阿里千问（深度绑定淘宝 / 支付宝 / 飞猪 / 高德的"AI 办事"）和腾讯元宝（依托微信生态做"社交智能体入口"），豆包**几乎完全独立于抖音 / 头条**——不依赖于现有 APP 内嵌，而是单独占据用户的 AI 助手入口
- **流量来源**：依赖**字节系应用矩阵的精准投放 + 春节红包大战**完成大规模拉新（一季度增量 1 亿）
- **场景定位**：从对话工具向"视频娱乐智能体入口"延伸，具备多模态生成能力；2025-10 起向"消费决策"场景拓展

> 来源：21 世纪经济报道 2026-02 综述、证券时报 2026-01-18 综述。注意这些是行业分析口径，非字节官方表态。

## 入口与可见性结构

### 用户主要触达豆包的方式

1. **独立 APP**（核心入口）
2. **Web 端 doubao.com**
3. **浏览器插件**（PC 场景）
4. **抖音内 AI 入口**（与抖音搜索 / 推荐协同）
5. **API（火山引擎）** —— 注意：API 访问与 APP 内访问的**信源结构有显著差异**（详见 [`citation-mechanism.md`](./citation-mechanism.md)）

### 与字节系生态的关系

- **抖音**：内容信源 + 用户路径
- **今日头条**：核心文章信源
- **抖音百科 / 快懂百科**：结构化知识信源
- **西瓜视频**：长视频信源
- **火山引擎**：模型与 API 出口

这种"模型 → 内容生态 → 用户分发"的全栈闭环是豆包区别于其他平台的关键结构性优势——在 GEO 视角下，意味着**字节系内部内容天然享有抓取与引用的优先级**（待实测验证具体权重）。

## 在仓库内的位置

豆包是仓库**第一个深度调研的 GEO 平台**，理由：

1. 月活第一（3.45 亿），用户基数最大
2. 字节生态闭环最深，是研究"内嵌生态如何影响 GEO 信源"的最好样本
3. 行业里围绕豆包 GEO 的服务商、研究、二手分析最丰富，但**质量良莠不齐**——值得仓库做权威整理

## 未解问题（待实测验证）

仅列出，**不在本卡片下结论**：

1. **字节系内部内容（头条 / 抖音 / 抖音百科）实际占豆包引用源的多少？**
   - 二手来源声称 "今日头条 60%+ + 抖音百科" → confidence: low，需自测
   - 不同 query 类型（资讯 / 决策 / 知识 / 创意）下的占比是否差异巨大？
2. **APP 内访问 vs API 访问的信源结构差异程度**
   - 二手来源声称差异显著 → 待用同 query 在两端跑实验对照
3. **时效性窗口**
   - 二手来源声称 "1-2 周内的内容权重最高" → 待用受控发布的内容跑时间序列实验
4. **是否有公开的"信源黑名单 / 白名单"机制？**
   - 完全未见官方说法，只能通过观察推断
5. **多模态信源（视频、音频）的转写与引用机制**
   - 抖音视频是否被转录后纳入文本检索？引用时呈现形式是什么？

候选实验题目（尚未排期）：

- `doubao-bytedance-ecosystem-bias`：对比同主题在字节系 vs 非字节系的内容引用率
