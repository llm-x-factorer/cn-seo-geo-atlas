# 千问引用机制（公开信息整理 + 待验证）

> ⚠️ **可信度声明**：与豆包 / DeepSeek / 文心卡片同样的限制——**没有官方文档背书，没有作者亲自实测**。所有"信源偏好"都来自第三方 GEO 服务商分析，应当被视为**假说**。
>
> 千问的特殊性：**接入电商交易场景**，使得"被千问引用"产生了"内容引用"和"商品调用"两种完全不同的优化目标。

## 研究边界

千问的入口比文心还多，必须先界定研究对象：

| 研究对象 | 数据来源 | GEO 优化的可控点 |
|----------|----------|------------------|
| **A. 千问 APP / chat.qwenlm.ai** | 千问模型 + 通用 RAG | 类似豆包 APP，目标是被对话式 AI 引用 |
| **B. 夸克搜索内的 AI 答案** | 千问 + 夸克索引 | **类似百度搜索 AI 答案位**，依赖夸克 SEO |
| **C. 淘宝 / 支付宝内嵌"AI 办事"** | 千问 + 阿里电商数据 | **新型 GEO**——SKU 推荐、商品页排名 |
| **D. 钉钉 AI（B 端协作）** | 千问 + 钉钉文档 | 企业内部场景 |
| **E. 阿里云百炼 API** | 千问模型 + 接入方 RAG | 完全由接入方决定 |
| **F. Qwen 开源模型自部署** | 仅模型训练数据 | 影响"未来训练数据"层 |

本卡片**主要讨论 A**（C 类电商场景另起独立卡片 `commerce-geo.md`）。

## 假说一：第三方资讯媒体偏好（与豆包 / 文心反例！）

> 来源：CSDN 博客《AI 搜索推荐机制深度拆解》（2026），泽森科技团队报告

声称的引用源权重排序（**未独立验证**）：

| 信源类型 | 声称权重 | 备注 |
|----------|----------|------|
| **网易号、搜狐号** | **60%+** | 千问的核心引用源 |
| 学术站、技术文档 | 高 | 专业 query 偏好 |
| 企业白皮书 | 中高 | 决策类 query |
| 第三方博客 | 中 | 社区 / 长尾 |
| **阿里自有生态（淘宝评价、飞猪笔记等）** | **未明确**——这正是关键反例 | 待实测确认是否被加权 |

**这是与豆包 / 文心 / 元宝的关键差异点**：

- 豆包：字节系内容（头条 / 抖音）声称占 60%+
- 文心：百度系内容（百家号 / 百科 / 知道）声称为核心
- **千问：第三方资讯媒体（网易号 / 搜狐号）声称占 60%+，自家电商生态内容地位未明**

**为什么这个假说特别值得验证**：

如果属实，它**反驳了"中文大厂 LLM 普遍偏好自家生态"的元假说**——至少阿里千问是个反例。

可能的原因（推测）：

- 阿里电商生态（淘宝评价、商品详情页）**结构上不像内容平台**——不是文章 / 笔记格式，难以直接被 RAG chunking 和检索
- 阿里把电商场景的"AI 调用"做成**专门的工作流**（C 类入口），而**通用对话场景**反而走通用 RAG 路径
- 因此千问 APP 的"通用对话"和"AI 办事"是两个独立链路——前者偏第三方资讯，后者直接调用阿里电商数据

**如果反驳成立的实操含义**：

- 想"被千问引用"≠ 在淘宝 / 飞猪上发笔记——电商内容对千问的通用对话引用可能影响有限
- 反而需要在**网易号、搜狐号、行业垂直媒体**布局——这与豆包优化路径完全不同
- 但电商类 query（"推荐 X""哪个品牌好"）会触发 C 类入口——又是另一套优化逻辑

**待办**：`exp-qianwen-citation-source-distribution`、`exp-qianwen-vs-aliyun-ecosystem`。

## 假说二：6 个月时效偏好（与豆包 / 文心 / DeepSeek 对比）

> 来源：CSDN 博客分析

| 平台 | 声称时效要求 |
|------|--------------|
| **豆包** | 1-2 周内权重最高 |
| **千问** | **6 个月内**，时间戳明确的内容优先 |
| 文心 | 3 个月内，首段含最新数据 |
| DeepSeek | 无明确期限，技术内容长期有效 |

**结构性解读**：

- 豆包最严（信息流偏好实时）
- 千问最宽（除 DeepSeek 外）—— 与"决策类 query 主导"的用户画像匹配
- 文心居中

**这条假说如果成立的实操含义**：

- 千问 GEO 内容更新节奏可以**半年级**——比豆包慢得多
- 评测 / 行业分析类内容在千问上的"半衰期"显著长于豆包
- 与"男性用户偏好深度内容"的用户画像匹配

## 假说三：表格 / 列表 / 模块化结构偏好

> 来源：CSDN 博客分析

声称千问偏好的内容结构：

- **表格**（产品对比、参数对比）
- **列表**（数字编号、要点罗列）
- **模块化写作**（清晰 H2/H3 分段）
- 整体风格：**客观中立、高信息密度、第三方报道口吻**

**结构性合理性**：

- 与"网易号 / 搜狐号" 的内容范式一致（资讯媒体偏好 + 客观叙述风格）
- 与"硬核参数" 用户画像匹配
- 是 LLM 通用结构偏好的中文具体化

**强信号**——这条相对可信，但具体阈值未知。

## 假说四：电商类 query 触发"AI 下单"工作流（C 类入口独立逻辑）

千问的**独特新型 GEO 问题**：

当用户问"推荐一双跑鞋"，千问的处理路径有两种可能：

| 处理路径 | 触发条件 | 信号源 |
|----------|----------|--------|
| **通用对话** | 一般"了解 / 比较"类 query | 通用 RAG（网易号 / 搜狐号等） |
| **AI 下单** | 明确"购买 / 哪里买"类 query | 直接调用淘宝 API + 商品数据 |

**对 GEO 实践的含义**：

- 这是**淘宝 SEO** 与 **千问 GEO** 的接合点——商品标题、详情页、评价、店铺信誉成为 GEO 信号
- 但与"内容 GEO"完全不同——不是优化文章，是优化电商商品页
- 详细机制留给独立卡片 [`commerce-geo.md`](./commerce-geo.md)（待建）

## 假说五：与四大厂的对照表

四个 GEO 平台（豆包 / DeepSeek / 文心 / 千问）的关键引用机制对照：

| 维度 | 豆包 | DeepSeek | 文心 | **千问** |
|------|------|----------|------|----------|
| **核心引用源** | 头条、抖音、抖音百科 | CSDN、搜狐号、网易号 | 百家号、百度百科、百度知道 | **网易号、搜狐号、学术 / 技术** |
| **生态偏好** | 强（字节） | 几乎无 | **最强（百度）** | **弱（与自家生态绑定不深）** |
| **时效偏好** | 1-2 周 | query 类型决定 | 3 个月 | **6 个月** |
| **结构偏好** | FAQ / 列表 / 信息流 | 代码块 / 架构图 | 首段定义 / 数据 / 总结句 | **表格 / 列表 / 模块化** |
| **品牌植入风格** | 信息流自然提及 | 技术实践者 | 权威解读者 | **第三方报道口吻、从业者视角** |
| **YMYL 表现** | 较弱 | 较弱 | 强 | 中（无医疗 / 金融垂直生态，但有学术偏好） |
| **电商 / 交易表现** | 中（接入抖音商城） | 弱 | 弱 | **极强（"AI 下单"全链路）** |
| **多模态** | 强（视频） | 弱 | 强（文心 5.0 全模态） | 中（Qwen-VL 等开源） |

**这张四大厂对照表的研究价值**：

- 如果四大厂引用机制差异显著 → "通用中文 GEO 策略"概念**彻底破产**，必须按平台分流
- **千问的"反例性"**（不偏好自家生态）尤为关键——它是元假说 B（生态内 > 生态外）的潜在反例

## 反假说 / 怀疑信号

调研中观察到的不一致信号：

1. **"网易号 / 搜狐号 60%+"未必跨 query 类型稳定**——技术 query 可能转向 CSDN，决策类可能转向第三方报告
2. **千问爆发式增长（一季度月活 +1.26 亿）**——大量"浅度用户"可能改变模型在线学习的反馈信号
3. **千问 APP vs 夸克 vs 淘宝内嵌 vs 钉钉**——服务商分析常笼统归为"千问"，实际行为差异可能巨大
4. **Qwen 开源版与千问 APP 闭源版**——可能完全不同的 RAG 配置、fine-tune 程度

## 在做实验前能下的结论

**可以下的（强）**：

- 千问月活第二（1.66 亿，一手数据）
- 千问与 Qwen 开源是**双轨结构**（事实）
- 千问 APP 在通用对话和电商交易场景下**走完全不同的工作流**（结构性事实）
- 千问对自家生态（淘宝 / 飞猪）的依赖度**显著低于豆包对字节系、文心对百度系**——这是仓库的关键观察

**不能下的**：

- 任何具体的"权重百分比"
- "网易号 60%+"等量化数字
- "AI 下单" 路径的精确触发条件
- 用户画像差异对引用偏好的实际影响程度
