# 千问平台概况

> ⚠️ **状态说明**：基于 QuestMobile 一手数据 + 公开战略报道 + 第三方 GEO 服务商二手分析，**未经作者亲自实测**。

## 为什么千问在仓库内是关键平台

千问是仓库**第六个深度调研的平台**——和豆包 / 文心 / DeepSeek 形成**四点 GEO 平台对照矩阵**：

| 维度 | 豆包（字节） | DeepSeek（独立） | 文心（百度） | **千问（阿里）** |
|------|--------------|------------------|--------------|--------------------|
| **公司类型** | 大厂 + 内容帝国 | 创业公司 | 大厂 + 搜索帝国 | **大厂 + 电商帝国** |
| **模型授权** | 闭源 | 开源 | 闭源 | **既闭源（千问 APP）也开源（Qwen 系列）** |
| **生态偏好** | 字节系（头条 / 抖音） | 几乎无 | 百度系（百家号 / 百科） | **阿里电商系 + 第三方资讯（网易 / 搜狐）** |
| **拉新方式** | 红包 + 字节系投放 | 模型出圈 | 嵌入百度搜索 | **春节红包 + 阿里生态投放 + 行业首创 "AI 下单"** |
| **场景定位** | 视频娱乐 + 消费决策 | 研究 / 编程 | 知识 / 生产力 | **消费 / 交易 / 生活管家** |

**千问的结构性独特性**：

1. **阿里电商生态深度绑定**——千问是国内**第一个把 AI 助手直接接入电商交易**的产品（淘宝、支付宝、淘宝闪购、飞猪、高德），AI 不仅"回答"还"办事"
2. **既闭源又开源**——Qwen 系列开源（与 DeepSeek 同列）+ 千问 APP 闭源——这是其他大厂没有的双轨结构
3. **核心引用源不在自家生态**——服务商分析声称千问偏好网易号、搜狐号等**第三方资讯媒体**而非阿里自有内容（如淘宝 / 飞猪笔记）。**这与豆包 / 文心的"自有生态优先"假说形成反例**

**关键反差**：

- 千问月活第二（1.66 亿），但**人均使用次数（19.8）远低于豆包（54.8）、DeepSeek（41.7），甚至低于元宝**
- **解读**：千问大量月活来自春节红包拉新 + 阿里生态投放；用户"装了但用得浅"——这是用户结构最浅的头部平台
- **对 GEO 实践的含义**：千问的"被引用次数"在中文 LLM 中实际权重可能低于其用户规模暗示的水平——同样规模的引用对豆包用户的曝光更深

## 战略定位

### "AI 办事"路线（独立 APP + 全生态打通）

千问选择了**独立 APP + 阿里生态全打通**的路线：

- **接入服务**：淘宝、支付宝、淘宝闪购、飞猪、高德、淘宝闪购
- **AI 能力**：不只"对话"还能"办事"——一句话点外卖、订机票、订酒店、买东西
- **战略意义**：阿里把现有的电商、本地生活、出行等业务优势**平滑转化为 AI 时代的用户黏性与数据资产**

行业评论：千问把"被搜索"和"被办事"打通——**搜索的下一个阶段不是看链接，而是 AI 直接完成任务**。

### 与豆包 / 文心 / 元宝的路线差异

| 大厂 | 路线 | 核心信号 |
|------|------|----------|
| 字节豆包 | 完全独立 APP | 从内容生态获取流量入口，不依附抖音 |
| **阿里千问** | **独立 APP + 阿里生态全打通** | **"AI 下单"——交易闭环** |
| 百度文心 | 嵌入式 + 多入口 | 利用百度搜索基本盘 |
| 腾讯元宝 | 独立 APP + 微信生态渗透 | 社交链 + 私域 |

### Qwen 开源生态（与 DeepSeek 共存的开源大厂）

阿里的**Qwen 系列开源模型**：

- HuggingFace 下载量长期位列中文模型 TOP1-2
- Qwen 2.5、Qwen3、Qwen3-Max 等持续迭代
- ModelScope（阿里魔搭）作为国内开源社区主入口

**对仓库的意义**：千问与 DeepSeek 都属"开源 + 闭源双轨"——但战略意图不同：

- **DeepSeek**：开源是核心战略（创业公司用开源打入主流）
- **阿里千问**：开源是生态策略（构建开发者生态 + B 端云服务变现），闭源 APP 才是 C 端核心

## 入口与可见性结构

### 用户主要触达千问的方式

1. **千问 APP**（核心 C 端入口）
2. **Web 端 chat.qwenlm.ai**
3. **夸克搜索**（阿里旗下，AI 搜索集成千问）
4. **钉钉 AI**（B 端协作集成）
5. **淘宝 / 支付宝内嵌**（"AI 办事"入口）
6. **百炼大模型平台 API**（开发者）
7. **Qwen 开源生态**（自部署 / 第三方接入）

### "AI 办事"路径下的可见性问题（独特）

千问 / 阿里电商场景下产生了一个**新型 GEO 问题**：

- 当用户问"推荐一双跑鞋"，千问可能不只回答品牌名，而是**直接调用淘宝 API 展示商品卡 + 一键下单**
- 此时"被引用"的不只是品牌词，而是**具体的淘宝 SKU**
- 这意味着 GEO 优化的对象**从内容扩展到电商商品页本身**——商品标题、详情页、评价数据、店铺信誉都成为 GEO 信号

> 这是其他平台没有的研究空间：**淘宝 SEO ≈ 千问 GEO（在交易场景下）**——这个等式如果成立，意味着电商品牌的 GEO 优化路径与传统淘宝运营深度重叠。

## 用户画像（QuestMobile）

千问 2026Q1 用户画像：

- **男性占比偏高**——服务商分析声称"硬核参数是关键之一"
- 春节红包拉新带来大量"浅度用户"
- 与豆包"银发 + 下沉" 画像不同——千问偏向"成熟决策型 + 一线 + 男性"

**对 GEO 内容的影响**（待验证假说）：

- 男性用户偏好场景下，"硬核参数 / 专业测评"内容可能更被偏好
- 与文心"权威解读"、豆包"信息流" 形成内容风格对照
