# 中文场景的"私域 vs 公域"信源光谱

> ⚠️ **状态：concept**（光谱分级 + 可达性差异是事实层观察，但具体引用率差异的量级未实证）
>
> 本卡补全前 10 张技术卡的另一个共同盲区——它们都默认信源的**公开度**已知或不重要。但中文场景的内容生态高度**"半公开化"**：英文圈典型 Web 是开放 HTML，中文场景的核心内容大量驻留在"登录后可见 / 注册后可见 / 平台代理后可见"的中间地带。
>
> 本卡做四件事：
>
> 1. 把"公开度"作为**5 档光谱**而不是公域 / 私域二分——中文场景的关键现象都在中间档
> 2. 横向对照各信源平台在光谱中的实际位置（同一平台不同内容档位可能不同——知乎是典型例子）
> 3. 拆解各档信源对中文 LLM 训练 / RAG 召回的可达性差异
> 4. 反"公众号矩阵覆盖所有 AI"话术——拆开"理论可达"和"实际被召回"
>
> ⚠️ **怀疑锚点 1**：公开度是**信源被 LLM 召回的必要条件之一**，不是充分条件——L1 完全公开的内容也可能因为权重低 / 域名垃圾 / 重复度高而不被 LLM 选用。本卡不暗示"提升公开度 = 提升引用率"。
>
> ⚠️ **怀疑锚点 2**：**"理论可达"≠"实际被召回"**。公众号文章在 mp.weixin.qq.com 域上是技术上可爬取的（L2），但**多数中文 LLM 厂商是否真的爬取它们 + 真的入库**完全不公开。"半公域"信源的实际 LLM 可见性是**结构性灰色地带**。
>
> ⚠️ **怀疑锚点 3**：[C-SEO Bench](https://openreview.net/forum?id=oTeixD3oZO) 在英文场景对各类"内容信号优化"的反向证据再次适用——单纯把内容从 L3 推到 L1（如把小红书图文搬到独立站）的边际效应**未必显著**，因为内容质量信号在跨平台时也会变化。

## 一、5 档公开度光谱

中文场景的信源公开度不是二分，而是连续光谱。把它分 5 档便于横向对照：

| 档位 | 标签 | 访问条件 | 中文场景代表 |
|------|------|----------|--------------|
| **L1** | 完全公域 | 任意客户端、无登录、静态 HTML 可爬 | 独立博客 / 政府站 / 维基中文（被墙但可下 dump）/ 学术开放期刊 |
| **L2** | 半公域（平台代理） | 可被外部爬取但 URL 限于特定平台域名 | 微信公众号文章（mp.weixin.qq.com） / 视频号 page / 部分知乎页 / 抖音视频页（含字幕）|
| **L3** | 注册可见 | 未注册看摘要 / 注册后看全部 | 小红书笔记完整内容 / B 站部分会员视频 / 知乎部分回答 |
| **L4** | 登录受限 | 必须登录到特定用户体系 | 微信朋友圈 / 私信 / 知乎私信 / 群聊 |
| **L5** | 全私域 | 企业 / 个人内部，无法外部访问 | 飞书 / 钉钉 / 企业微信 / 内部 Wiki / 私有云笔记 |

### 1.1 关键差别 vs 英文场景

| 维度 | 英文场景 | 中文场景 |
|------|---------|----------|
| 主流内容载体 | 独立站为主（L1） | "号"为主（L2 占大头） |
| 平台代理化 | 较弱 | **极强**——核心内容驻留在平台域名下 |
| L3 注册可见占比 | 较低 | **较高**——小红书 / 知乎 / B 站均有大量内容默认 L3 |
| L4-L5 比例 | 较低（企业知识库各国类似） | 类似（企业类无显著差异） |

**关键观察**：中文场景**L2 半公域**占核心信息池的比例**显著大于**英文场景。这意味着英文圈"做独立站"思维在中文场景需要彻底重新设计——大量优质内容根本不在独立站。

## 二、各信源平台的光谱位置

> ⚠️ **同一平台的不同内容档位可能不同**。本表给出**主流内容**所在档位，并标注例外。

| 平台 | 主档位 | 例外 | 备注 |
|------|--------|------|------|
| 独立博客 / Hexo / WordPress | L1 | 部分私密文章 L4 | 中文独立博客生态萎缩多年 |
| 政府站 / `*.gov.cn` | L1 | 部分内部文档 L5 | YMYL 类信源 |
| 学术 / `*.edu.cn` / 知网 | L1 + L3 | 知网论文需付费 L3 | 国内学术论文付费墙较强 |
| 维基中文 | L1（dump 公开） | 大陆访问受限 | 详见 [`knowledge-graph-cn.md`](./knowledge-graph-cn.md) 元假说 E |
| 微信公众号 | **L2** | 阅读权限私域文章 L4 | 中文场景信源量级最大池之一 |
| 微信视频号 | L2 | 私密视频 L4 | 与公众号同生态 |
| 知乎 | L1 + L3 | 私信 L4 | 横跨多档——"开屏前看摘要"是设计 |
| 小红书 | **L2 + L3** | 仅自己可见笔记 L4 | 图文双轨 + OCR 路径见 [`multimodal-cn.md`](./multimodal-cn.md) |
| B 站 | L1（视频页） + L3（部分会员） | 私密 L4 | 弹幕 + 字幕作为多模态信源 |
| 抖音 | L2 | 私密 L4 | 视频字幕主要 LLM 可达路径 |
| 微博 | L1 + L2 | 部分博主仅互关 L3-L4 | "黑名单 / 仅好友"机制让公开度异质化 |
| 大众点评 / 美团 | L1 + L3 | 用户评价后看全部 L3 | 本地生活类查询特殊性 |
| 淘宝 / 京东 | L2 | 商品详情图为主 | 电商类查询的核心信源（[`platforms/qianwen/`](../platforms/qianwen/)） |
| 飞书 / 钉钉 / 企业微信 | L5 | 部分公开文档 L1 | 企业知识库 GEO 起点 |

## 三、各档信源对中文 LLM 的实际可达性

> ⚠️ **本节多为推测**——5 大中文 LLM 厂商均未公开数据采集策略与各档信源的覆盖度。

### 3.1 训练数据爬取（路径 1）

| 档位 | 通用爬取可达性 | 关联 LLM |
|------|--------------|----------|
| L1 | **高**——常规爬虫即可 | 所有 LLM 大概率覆盖 |
| L2 | 中—不一致——需要应对平台反爬 + 渲染 | 生态内 LLM 应有内部路径（豆包对抖音、元宝对公众号、文心对百家号、千问对淘宝） |
| L3 | **低**——需要登录态，多数通用爬虫不做 | 各生态内 LLM 可能用自家账号爬取，跨生态弱 |
| L4 | **极低**——基本不可达 | 几乎无 |
| L5 | **0**——技术上不可达 | 几乎无（除非主体主动开放） |

### 3.2 RAG 实时检索（路径 2）

| 档位 | RAG 可达性 | 关联 LLM |
|------|-----------|----------|
| L1 | 高 | 所有 LLM |
| L2 | 与训练同——生态内 LLM 占优 | 同 3.1 |
| L3 | 几乎不可达——RAG 实时不便携带登录态 | 几乎无 |
| L4-L5 | 0 | — |

### 3.3 反向特例：用户上传 / 转述路径（路径 3）

L3-L5 档信源**通过用户行为间接进入 LLM**：

- 用户截图 / 转述 L3 内容（如小红书笔记），让 LLM 回答
- 用户把 L4-L5 内部文档粘到 LLM 进行总结
- 这些信号**不进入 RAG 召回 / 训练数据**，但**可能影响 LLM 输出**

**关键观察**：路径 3 是**用户侧**而非内容方控制——内容方对其几乎无影响。但它解释了一个常见现象：私域内容看似"不可见"，但通过用户行为不断进入 LLM 的对话上下文，形成"似无似有"的传导。

## 四、各档信源对中文搜索引擎的可达性

| 档位 | 搜索引擎可达性 | 备注 |
|------|--------------|------|
| L1 | 高（已备案前提下） | [`icp-and-subject-cn.md`](./icp-and-subject-cn.md) 第三节 |
| L2 | **平台决定**——微信公众号在百度长期不可搜（仅微信搜一搜可搜）；公众号对外有内容 SEO 文章但量级有限 | 各生态有"信源协议"门槛 |
| L3 | 摘要可见，全文不可见 | 用户体验受损 |
| L4-L5 | 几乎不可达 | — |

**核心模式**：搜索引擎对 L2 半公域的处理是**平台谈判结果**——微信对百度的封锁、抖音对外的部分开放等都是商业决策结果。这意味着 L2 信源对中文搜索引擎的可达性是**人为可变**的，不是技术问题。

## 五、待验证元假说

### 元假说 A：中文 LLM 召回的信源以 L1 完全公域为主——但生态内 LLM 显著拉高 L2 占比

- 表述：DeepSeek / Kimi 等通用 LLM 召回信源中 L1 占比 ≥ 70%；豆包 / 元宝 / 文心 / 千问 等生态内 LLM 召回信源中 L2 占比可能达 30-50%
- **驱动力**：生态内 LLM 有特殊路径访问自家 L2 信源
- **如何验证**：在 [`exp-five-vendor-paired-queries`](../experiments/exp-five-vendor-paired-queries.md) 数据中按信源域名分类，统计各档占比
- **关联**：[`eeat-cn.md`](./eeat-cn.md) 元假说 B（生态作为权威性代理）的公开度侧具体化

### 元假说 B：公众号矩阵的 GEO 收益**仅在元宝 / 微信搜一搜显著**

- 表述：发布 N 篇主题对齐的公众号文章后，**只有元宝 / 微信搜一搜的引用率显著提升**；豆包 / DeepSeek / 文心 / 千问 / Kimi 几乎不响应
- **驱动力**：腾讯生态闭环——公众号 → 元宝 / 微信搜一搜
- **如何验证**：受控发布 + 跨 5 平台对照
- **关联**：[`account-grade-systems-cn.md`](./account-grade-systems-cn.md) 元假说 B（账号等级仅在生态内传导）的信源载体侧
- **含义**：服务商话术"公众号矩阵覆盖所有 AI"在 4/5 平台上是话术，仅 1/5 成立

### 元假说 C：L3 注册可见信源的 GEO 路径**主要靠用户截图 / 转述**而非平台爬取

- 表述：小红书 L3 笔记（注册后看全部）的内容进入中文 LLM 的主路径不是直接爬取，而是用户**复制粘贴 / 截图询问**
- **驱动力**：登录态难传递 + 平台反爬 + 反向 OCR 用户路径反而打通
- **如何验证**：观察 LLM 对"小红书笔记关键词"的响应深度；如果只能给摘要，说明走的是公开摘要路径；如果能给详尽内容，可能走了某种全文获取路径
- **关联**：[`multimodal-cn.md`](./multimodal-cn.md) 元假说 D（图里文字 > alt）的姊妹假说

### 元假说 D：L4-L5 全私域内容是企业知识库 GEO 的"零信号"基线

- 表述：飞书 / 钉钉 / 内部 Wiki 等 L5 内容**对所有中文 LLM 引用率为 0**——除非主动开放
- **含义**：企业 GEO 的第一步**不是优化**，是**"决定哪些内容公开"**——这是商业决策，不是技术决策
- **副作用**：服务商"提升企业内部内容 AI 可见性"必须先经过"公开度调整"环节，绕过这一步的承诺都是话术

### 元假说 E：把内容从 L2-L3 迁移到 L1（独立站）的边际收益**未必显著**

- 表述：把已发在公众号 / 小红书的内容搬到独立站后，并不必然提升中文 LLM 引用率——因为：(1) 独立站本身需要域名权重 + 备案 + SEO 收录; (2) L1 公域信源池竞争激烈，新站默认排序极低
- **含义**：内容方**不能简单地通过"提升公开度"提升 GEO**——还需要跨过"L1 池中获得权重"的门槛
- **副作用**：服务商"把内容搬到独立站提升 AI 引用"是低估了 L1 池的竞争强度

## 六、实验设计建议

### 实验 1：按档位归类信源后的 LLM 引用率分布（核心实验）

- **目标**：验证元假说 A
- **设计**：
  - 数据来源：[`exp-five-vendor-paired-queries`](../experiments/exp-five-vendor-paired-queries.md) 跑出来后，扩展 [`tools/exp-doubao-vs-deepseek-paired/source_classifier.py`](../tools/exp-doubao-vs-deepseek-paired/source_classifier.py) 加入 L1-L5 档位标签
  - 因变量：每个 LLM 召回信源的 L1-L5 档位分布
- **复用**：直接套用现有工具栈
- **优先级**：⭐⭐⭐ 高（无需新数据采集）

### 实验 2：公众号矩阵跨 5 平台 LLM 引用率对照（验证元假说 B）

- **目标**：直接证伪 / 验证服务商"公众号矩阵覆盖所有 AI"话术
- **设计**：
  - 自变量：2 档（同主题在 5 个公众号同时发 vs 仅发独立站）
  - 因变量：5 大中文 LLM 引用率
  - 控制：内容质量、长度、发布时间
- **样本**：每档 ≥ 30 主题 × 30 天
- **优先级**：⭐⭐⭐ 高（直接打脸高频服务商话术）

### 实验 3：L3 注册可见内容的传导路径观察（验证元假说 C）

- **目标**：观察小红书 L3 笔记是否真的不能直接被 LLM 召回
- **设计**：选择**仅在小红书发布且未被搬运**的笔记内容（控制独占性），让 5 大 LLM 回答与笔记内容相关的 query，观察是否能给出笔记中的特定细节
- **预期**：如果回答只能给公开摘要级信息，元假说 C 部分成立；如果能给详尽内容，则有未公开的 L3 爬取路径
- **难点**：需要确认笔记内容确实"独占"——爬取检测站外是否有镜像
- **优先级**：⭐⭐ 中

### 不建议做的实验

- **"把所有公众号文章都搬到独立站"对照实验**：内容质量信号在跨平台时也会变化，归因困难（元假说 E 的难点）
- **"在飞书 / 钉钉企业内部知识库做 SEO"**：L5 信源对外几乎不可达，对照组都不存在
- **"做 N 个公众号同名矩阵"**：进入服务商灰产边界，仓库不收

## 七、与 GEO 服务商话术的对照

国内常见话术：

- "公众号矩阵 = AI 引用率多倍提升"
- "把内容搬到独立站，被所有 AI 收录"
- "私域内容也能做 AI 优化"
- "覆盖所有公开度档的内容池，AI 必引"

**本卡片判定**：

| 话术 | 判定 | 理由 |
|------|------|------|
| "公众号矩阵 = AI 引用率多倍提升" | ❌ 仅元宝 / 微信搜一搜成立 | 元假说 B；其他 4 家 LLM 几乎不响应 |
| "搬独立站，所有 AI 收录" | ❌ 低估 L1 池竞争 | 元假说 E；新独立站需要权重积累 |
| "私域内容做 AI 优化" | ❌ 类别错误 | 元假说 D；L4-L5 信源是 0 信号，必须先公开 |
| "覆盖所有公开度档" | ⚠️ 措辞模糊 | 各档对各 LLM 的可达性不同——"覆盖"不等于"被引用" |
| "在生态内做 L2 半公域是中文 GEO 核心" | ✅ 与元假说 A + B 部分一致 | 中文场景 L2 占核心池显著大于英文 |

**结论**：服务商话术里**与"在对应生态内做 L2 半公域"对齐**的部分相对靠谱；**承诺"覆盖所有 LLM"或"私域也能 GEO"**的部分是话术。

## 八、与本仓库其他卡片的关系

- 与 [`account-grade-systems-cn.md`](./account-grade-systems-cn.md)：账号等级体系与公开度光谱是**两个正交维度**——L1-L5 是**信源访问性**，L1-L3 账号等级是**主体合规性**。两者结合可拆得更细
- 与 [`icp-and-subject-cn.md`](./icp-and-subject-cn.md)：ICP 备案是 L1 完全公域 + 可被搜索引擎收录的前置条件
- 与 [`llms-txt-cn-practice.md`](./llms-txt-cn-practice.md)：llms.txt 假定信源在 L1（自有站点根目录）；L2 半公域信源（公众号）无法部署 llms.txt
- 与 [`citation-attribution-styles-cn.md`](./citation-attribution-styles-cn.md)：本卡元假说 B 与该卡元假说 C（元宝公众号软回流）是同一现象的两个侧面——前者讲"被召回的概率"，后者讲"被召回后呈现的形态"
- 与 [`multimodal-cn.md`](./multimodal-cn.md)：L3 注册可见信源（如小红书图文）的多模态特征使其有用户截图 / OCR 的反向路径
- 与 [`platforms/yuanbao/citation-mechanism.md`](../platforms/yuanbao/citation-mechanism.md)：公众号 → 元宝 是 L2 半公域 → 生态内 LLM 的代表案例

## 九、不做的事

- **不做"如何把私域内容公开化"操作教程**——本仓库不收录平台运营技巧
- **不评判"哪种公开度档最优"**——取决于商业目标 + 主体类型
- **不预先断言中文 LLM 的爬取深度**——所有厂商都未公开
- **不进入"账号矩阵代运营"灰产价格 / 渠道讨论**——本仓库不收
- **不为单一公开度档给"权重系数"**——平台 + LLM 联合决定，无法独立给系数
