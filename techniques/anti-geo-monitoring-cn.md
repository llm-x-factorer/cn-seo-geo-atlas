# 反 GEO 监测：防御侧赛道与对称市场拆解

> ⚠️ **状态：concept**（基于 DS 维度元假说 C 的具体展开 + 行业观察 + SEO 行业类比 + 探词手册"舆情监控与负面评价管理"产品持续优化方向佐证。**无任何中文场景大样本独立调研**——本卡是结构判断，不是市场报告）
>
> 这张卡是 [`geo-business-model-taxonomy-cn`](./geo-business-model-taxonomy-cn.md) **DS 元假说 C** 的具体展开——展开"反 GEO 监测是被严重低估的对称赛道"在客户类型 / 需求结构 / 技术壁垒 / 商业模式 / 监管驱动 5 个维度的具体含义。
>
> 本卡做四件事：
>
> 1. 把"反 GEO 监测"按需求类型拆为 4 类，避免"AI 舆情监测"这个总称混淆不同付费心理
> 2. 给出**与传统舆情监测的对照**——技术差异决定为什么现有舆情厂商没有自然延伸到该赛道
> 3. 拆**客户决策人 / 预算来源 / 购买心理**——这与进攻型 GEO 完全不同（公关 / 法务 vs 营销）
> 4. 识别**监管驱动**的合规子赛道（YMYL）——不依赖市场教育的强需求
>
> ⚠️ **怀疑锚点 1**：[C-SEO Bench](https://openreview.net/forum?id=oTeixD3oZO) 严格控制下进攻型 GEO 多数无效——但**监测有效性独立于优化有效性**。"做 GEO 没用"和"监测 GEO 现状有用"不矛盾，反而互相加强（越是优化无效的市场，知情成本越高）
>
> ⚠️ **怀疑锚点 2**：本卡判断"反 GEO 监测需求大"主要基于**类比推理**（传统舆情 / 负面 SEO 监测 / 商标监测的成熟剧本），而非中文场景独立证据。元假说 A 标⭐⭐⭐ 但需访谈实证后才能升 confirmed
>
> ⚠️ **怀疑锚点 3**：**LLM 厂商可能自建"品牌方监测后台"**（类似 Google Search Console for AI）——这是反 GEO 监测的最大单一平台风险，本卡第 IV 元假说 G 专门讨论

## 一、概念澄清

### 1.1 进攻型 GEO vs 防御型 GEO 监测

两个完全对称的赛道，**客户重叠度低**：

| 维度 | 进攻型 GEO | 防御型 GEO 监测 |
|------|-----------|---------------|
| 目的 | 让自家在 LLM 答案中**出现 / 排前 / 被推荐** | 让自家**不被误述 / 不被攻击 / 不被舆情绑架** |
| 客户决策人 | 营销部 / 市场增长部 | **公关部 / 法务部 / 舆情监测部** |
| 预算来源 | 市场预算 | **公关预算 / 法务预算 / 合规预算** |
| 购买心理 | 投资 / ROI 期待 | **风险厌恶 / 知情权 / 合规义务** |
| 效果衡量 | 引用率 / 位次提升（难） | 监测覆盖率 / 报告完整度（易） |
| 是否有反事实问题 | 严重（"不做也可能这样"） | 轻微（"知情" 不要求改变结果） |

**关键差异**：防御型客户买的是**知情权**，不是**效果**——这绕开了进攻型 GEO 最致命的"效果不可独立验证"问题。

### 1.2 反 GEO 监测的 4 类需求

按 LLM 答案体可能造成的"问题类型"拆：

| 类型 | 监测对象 | 客户痛点 | 客户类型 | 付费意愿 |
|------|---------|---------|---------|---------|
| **N1 品牌可见度被动监测** | 自家被 LLM 怎么描述（含正 / 负 / 错） | 不知情焦虑 + 公关响应 | 中大型品牌公关 | 中-高 |
| **N2 竞品攻击监测** | 竞品在 LLM 答案中是否"踩"自家 | 竞品截流 + 比较型话术 | 营销 + 法务 | 中 |
| **N3 AI 幻觉监测** | LLM 编造的虚假信息（不存在的产品 / 错误的价格 / 假冒高管姓名 / 错误的法律事实） | 法律风险 + 监管合规 | 法务 + 合规（YMYL 主战场） | **高**（强需求） |
| **N4 AI 舆情扩散监测** | 负面舆情通过 LLM 二次传播 | 危机扩散 + 二次发酵 | 公关 + 危机管理 | 高 |

注：4 类的客户类型有重叠，但付费心理和决策路径不同。多数现有"AI 舆情监测"产品只覆盖 N1，N2-N4 的供给几乎空白。

### 1.3 与传统舆情监测的边界

很多读者会问：传统舆情监测厂商（蜂鸟 / 海纳 / 鹰眼 / 慧科 等）不就能延伸做这个吗？

| 维度 | 传统舆情监测 | 反 GEO 监测 |
|------|------------|-----------|
| 监测对象 | 公开网页 + 社交媒体 + 新闻 + 论坛 | **LLM 答案体**（动态生成） |
| 数据获取方式 | 爬虫 + RSS + API + UGC 抓取 | **模拟用户 query 拿答案体**（每次都新生成） |
| 是否有 URL 锚点 | 有（每条数据可定位） | **无**（答案体不可定位 / 不可引用） |
| 复测性 | 同 URL 同时刻数据相同 | **低**（同 query 不同账号 / 时段答案不同） |
| 时间衰减 | 内容发布后稳定 | **每次新生成，无历史**（除非自己存） |
| 个性化干扰 | 几乎无 | **强**（用户画像 / 历史会话影响答案） |
| 量级 | 海量但稀疏 | **窄但密集**（每个 query × N 平台 × M 重复 × K 账号） |

技术差异决定了**传统舆情厂商无法自然延伸**——爬虫栈不能直接复用，监测方法学要重建。这是反 GEO 监测的技术壁垒所在。

## 二、横向对照表

### 2.1 4 类需求 × 5 类客户 × 付费意愿 × 当前供给

| 客户类型 | N1 品牌可见 | N2 竞品攻击 | N3 AI 幻觉 | N4 AI 舆情扩散 |
|---------|-----------|-----------|-----------|-------------|
| **大型品牌公关部** | 强 / 部分供给 | 中 / 空白 | 中 / 空白 | 强 / 空白 |
| **法务部 / 合规部** | 中 | 中-强 / 空白 | **强 / 空白** | 中 |
| **YMYL 类持牌主体**（医药 / 金融 / 法律） | 中 | 中 | **强 / 空白**（监管驱动） | 中 |
| **上市公司董秘 / 投关** | 中 | 弱 | **强**（信息披露合规） | 强 / 空白 |
| **中小品牌主** | 弱（预算不够） | 弱 | 弱 | 弱 |

**结论**：N3（AI 幻觉监测）+ N4（AI 舆情扩散）+ N1（品牌可见度，未供给的部分）是**强需求 × 空白**的双重交叉。其中 N3 在 YMYL 类是**监管驱动的刚需**——不依赖市场教育。

### 2.2 各 LLM 平台监测难度对照

| 平台 | 反爬难度 | 答案稳定性 | 个性化干扰 | 跨账号复现成本 | 总监测难度 |
|------|---------|-----------|-----------|--------------|----------|
| 豆包（Web） | 中（字节系反爬强） | 中 | 中-强（推荐系统耦合） | 高 | 高 |
| 千问（Web / APP） | 中 | 中 | 中 | 中 | 中-高 |
| 文心（Web） | 低-中 | 中 | 中 | 中 | 中 |
| 元宝（Web / APP） | 中 | 中 | 中 | 中 | 中-高 |
| Kimi（Web） | 低 | 中-高 | 低-中 | 低-中 | 中 |
| DeepSeek（Web） | 低 | **高** | 低 | 低 | **低**（最易监测） |
| 百度 AI（搜索内嵌） | 低 | 低（与搜索结果耦合） | 低 | 低 | 低 |
| 小红书问一问 | 中（微信 / APP 内 + 反爬） | 中 | 强 | 高 | 高 |
| 微信 AI 搜（公众号 / 视频号） | **高**（强账号绑定） | 中 | 强 | **极高** | **高** |

**结论**：DeepSeek + 百度 AI 是反 GEO 监测的最低成本起点；微信 AI 搜 + 豆包 + 小红书是技术壁垒最高的目标——**跨平台一致性监测**的技术壁垒主要在这三个。

### 2.3 反 GEO 监测 vs 进攻型 GEO 在 [`geo-business-model-taxonomy-cn`](./geo-business-model-taxonomy-cn.md) 七层中的位置

| 层 | 进攻型典型玩家 | 反 GEO 监测对应位置 |
|---|--------------|------------------|
| a 数据 / 监测 | 探词 APIaaS / 监测 SaaS | **直接落在 a 层**——监测数据 + 报告 + 告警 |
| b 工具 / SaaS | 垂直工具 | 反向工具：幻觉率检测器 / 跨 LLM 一致性指数器 |
| c 内容 / 信源 | 7 万+ 媒体发布 | **几乎不涉及**——反 GEO 不做内容生产 |
| d 服务 / 咨询 | RaaS 代运营 | 防御陪跑（公关 + 法务联合服务） |
| e 平台 / 生态 | OEM 给同行 | 不直接对应 |
| **f 反向 / 防御** | — | ✅ **本卡所在层**——是 f 层的具体展开 |
| g 标准 / 协议 | llms.txt 推动 | 推动"AI 内容溯源 / AI 幻觉披露" 标准的衍生 |

**核心**：反 GEO 监测 ≈ DS 七层中的 a + f 层组合，**不需要 c/d 层**——这是它毛利结构远优于进攻型 GEO 的根本原因。

## 三、元假说集

### 元假说 A（强反话术，⭐⭐⭐）—— 反 GEO 监测的真实需求规模与进攻型 GEO 相当甚至更大

进攻型 GEO 的客户付费意愿要靠"AI 必投"市场教育激发；反 GEO 监测的客户付费意愿**已有舆情预算锚点**——大型企业的舆情监测年预算（百万级到千万级）已存在，把"AI 答案中的品牌呈现监测" 加进现有服务包是顺势扩展。

支持证据：
- SEO 时代对称剧本：负面 SEO 监测 + 域名抢注监测 + 品牌词保护服务，这些防御型业务规模不亚于进攻型 SEO
- 中国上市公司 / 大型企业已有舆情监测部门 + 年度预算
- 探词手册 p19 把"舆情监控与负面评价管理"列为产品持续优化方向之一——侧面印证厂商已感受到该需求

强反话术：
- ❌ "GEO 服务商之间竞争"（忽视防御侧客户）
- ❌ "客户都是营销部门买单"（防御型客户是公关 + 法务）

可证伪性：F——通过访谈 ≥ 10 家上市公司公关 / 法务 / 舆情监测总监验证需求强度 + 现有支付能力。

### 元假说 B —— 客户决策人是公关 + 法务而非营销，销售路径完全不同

进攻型 GEO 的销售路径：营销 / 市场总监 → POC → 投放 → 看效果。
反 GEO 监测的销售路径：公关 / 法务总监 → 风险评估 → 采购 → 看报告。

**两条销售路径不能复用**：

- 客户决策周期：营销看季度，公关 / 法务看年度（合规预算编制）
- 决策依据：营销看 ROI，公关 / 法务看**风险案例 + 合规要求**
- 续费驱动：营销看续期 KPI，公关 / 法务看**合规延续 + 危机案例发生**

强反话术：
- ❌ "GEO 服务商一站式覆盖进攻 + 防御"——多数厂商的销售团队只懂一种语言

可证伪性：P——通过对比已有玩家（探词等）的销售岗位 JD 和服务案例分布，间接验证销售路径分化。

### 元假说 C（强反话术，⭐⭐⭐）—— 反 GEO 监测的技术门槛比进攻型低

反 GEO 监测**只需 a 层（数据 / 监测）+ f 层（防御工具）**，不需要：

- c 层（内容生产 / 媒体投放）—— 防御型不做内容
- d 层（效果代运营）—— 防御型不承诺效果
- e 层（OEM 白标）—— 防御型客户不需要工具私有化

技术上需要：

- 跨 LLM 答案体抓取（与进攻型监测共用基础设施）
- 答案体语义识别（NER + 情感分析 + 幻觉检测）
- 跨平台一致性指数计算
- 报告 + 告警系统

支持证据：
- 多数能力已在传统舆情监测厂商技术栈中存在（除 LLM 抓取层）
- 监测系统的可标准化程度远高于代运营
- 单客户毛利天花板由 SaaS 订阅模型决定，不是人力天花板

强反话术：
- ❌ "GEO 全栈服务"（多数 GEO 厂商所谓"全栈"是为了报价高，技术上反而难做精）
- ❌ "我们既做进攻又做防御"（两个赛道的销售 + 技术 + 客户成功路径完全不同）

可证伪性：P——通过对比反 GEO 监测和进攻型 GEO 的研发投入 / 工程师配比，间接验证技术门槛差异。

### 元假说 D —— AI 幻觉监测会成为 YMYL 行业的合规必需，监管驱动而非市场驱动

YMYL（医疗 / 法律 / 金融）的 AI 幻觉具有**直接合规风险**：

- 医药：LLM 编造的"X 药能治 Y 病"造成医疗事故可能引发药企责任
- 法律：LLM 给出错误法律建议可能被律所被告
- 金融：LLM 编造的"X 公司财报"可能引发证券误导
- 上市公司：LLM 答案与公开披露不一致可能违反信息披露规则

《生成式人工智能服务管理办法》（2023-07-13）已落地，**后续细则逐步压实"AI 内容真实性 / 来源可追溯" 责任**——监管驱动的需求不依赖市场教育。

强反话术：
- ❌ "AI 幻觉是技术问题，等技术成熟就好"（监管不会等技术成熟，且 LLM 幻觉是结构性问题）

可证伪性：F——通过监测后续监管细则的发布 + 行业被罚案例，验证合规驱动的强度。

### 元假说 E —— 单 LLM 监测易，跨 LLM 一致性监测难

单平台监测的技术门槛低（DeepSeek / Kimi / 百度 AI 都易爬），但客户**真正想要的**是"跨 LLM 我家被怎么描述"——跨 LLM 一致性监测涉及：

- 不同 LLM 的答案语义对齐（同一事实在不同 LLM 中的表述差异）
- 时间窗口对齐（不同 LLM 的更新频次不同）
- 个性化对齐（账号 / 设备 / 时段差异）

技术壁垒主要在**语义对齐 + 个性化解耦**，这是 a 层在反 GEO 场景下的核心创新点。

可证伪性：F——可以直接构建跨 LLM 答案一致性指数（Jaccard / 编辑距离 / NLI），测量同 query 跨 6 平台的差异。

### 元假说 F —— 反 GEO 监测的"反事实基准"问题轻于进攻型

进攻型 GEO 的核心痛：客户问"我没花钱也可能本来就排第一" → 效果不可独立验证。

反 GEO 监测**绕开这个痛**——客户买的是"知情权"，不是"改变结果"。即使 LLM 答案没变，监测报告本身仍有价值（合规留痕 / 风险预警 / 内部决策支持）。

可证伪性：P——通过对比两种业务的客户续约率 / 投诉率，验证反事实问题的影响差异。

### 元假说 G —— LLM 厂商可能自建"品牌方监测后台"是反 GEO 监测的最大单一平台风险

类比 Google Search Console for SEO，LLM 厂商可能上线"品牌方查询自家在 LLM 中表现"的官方后台。一旦上线：

- N1 品牌可见度监测被官方后台部分替代（但跨平台仍需第三方）
- N2 / N3 / N4 大概率仍需第三方（厂商不会主动暴露竞品攻击 / 自家幻觉）
- 第三方独立性是核心壁垒

支持证据：
- Google Search Console 上线后并未消灭第三方 SEO 工具（Ahrefs / Semrush 至今活跃）
- 客户需要**跨平台对比** + **独立第三方报告**作为合规留痕

强反话术：
- ❌ "等 LLM 厂商自建后台第三方就死了"——SEO 工具行业 20 年历史已反驳此命题

可证伪性：F——通过监测豆包 / 元宝 / 千问 / 文心 等是否上线品牌方后台，验证替代风险。

## 四、3 个实验设计建议

### E1 ⭐⭐⭐ 防御侧客户访谈（公关 / 法务总监）

**目的**：验证元假说 A、B——反 GEO 监测的需求强度 + 销售路径分化。

**方法**：
- 招募 ≥ 10 名上市公司或大型企业的公关 / 法务 / 舆情监测总监
- 半结构化访谈，覆盖：
  - 当前 AI 答案中的品牌呈现是否在监测视野内
  - 当前 AI 幻觉 / 舆情扩散事件是否已发生（举具体案例）
  - 当前预算分配（舆情 vs AI 监测）
  - 决策路径（公关 / 法务谁主导）
  - 对"反 GEO 监测" 服务的付费意愿（具体范围）
- 输出：10 份匿名访谈记录 + 需求强度 + 预算规模 + 决策路径分析报告

**预算**：人力 ≈ 30-50 小时（联系 + 访谈 + 整理），现金 ¥0.5-3 万（访谈酬劳）

**预期产出**：
- 元假说 A 的实证（提升 confidence 从 low → medium）
- 给后续 ⭐⭐⭐ 反 GEO 监测专项卡 / 业务方向提供需求侧基准

### E2 ⭐⭐ YMYL vs 非 YMYL 类 LLM 幻觉率对照

**目的**：验证元假说 D 的合规驱动假设——YMYL 类 LLM 幻觉率是否系统性高于 / 低于非 YMYL 类。

**方法**：
- 设计 ≥ 100 个 query，分 YMYL（医疗 / 法律 / 金融，每类 20 个）和非 YMYL（生活 / 工具 / 学习，共 40 个）
- 在 6 大 LLM（豆包 / 元宝 / 千问 / 文心 / Kimi / DeepSeek）跑每个 query × 3 次
- 人工标注每个答案是否含幻觉（虚假事实 / 不存在的产品 / 错误的引用）
- 对比 YMYL 类 vs 非 YMYL 类的幻觉率

**预算**：人力 ≈ 40-60 小时（标注是主要成本），可与 [`tools/exp-doubao-vs-deepseek-paired/`](../tools/exp-doubao-vs-deepseek-paired/) 数据采集流程复用

**预期产出**：
- YMYL 幻觉率基线
- 给监管驱动假设提供量化支持

### E3 ⭐⭐ 跨 LLM 答案一致性指数测量

**目的**：验证元假说 E 的技术壁垒——跨 LLM 一致性监测的实际难度。

**方法**：
- 选 50 个跨平台 query，在 6 平台跑 × 5 次
- 对每个 query 跨平台答案做 Jaccard 相似度 + 编辑距离 + NLI（自然语言推理）三种一致性度量
- 输出：50 query × 3 指数 × 6 平台对照矩阵

**预算**：人力 ≈ 20-30 小时（采集脚本可复用）

**预期产出**：
- 跨 LLM 一致性指数的具体值
- 反 GEO 监测产品设计的技术参数基准

### 不建议做的实验

- ❌ "反 GEO 监测客户付费意愿问卷"——抽象问卷数据噪声大，应做 E1 半结构化访谈
- ❌ "幻觉总量调研"（不分行业 / 不分场景）——总量数字与商业价值无强因果
- ❌ "反 GEO 监测公司估值排名"——多数公司未成立或未融资，无可比数据

## 五、与 GEO 服务商话术的对照

| 服务商话术 | 本卡判定 |
|----------|---------|
| "我们提供 AI 舆情监测" | ⚠️ 多数其实是传统舆情 + AI 关键词；真做反 GEO 监测还需 LLM 答案体抓取 + 一致性指数 + 幻觉检测 |
| "我们能保证 LLM 不出现负面" | ❌ 不可保证。LLM 答案是合成的且个性化，**没有任何厂商能"删除" LLM 答案中的内容** |
| "AI 幻觉可控" | ❌ 厂商不能控制 LLM 输出，只能监测 + 告警 |
| "我们覆盖 N+ AI 平台幻觉监测" | ⚠️ 监测可，干预多数不可 |
| "GEO 服务商一站式（进攻 + 防御）" | ⚠️ 多数厂商的销售 + 技术 + 客户成功路径只擅长一种 |
| "做 GEO 优化降低幻觉率" | ❌ 类别错误。优化是让自家被引用 / 排前；幻觉是 LLM 编造内容，两件事 |
| "我们做反 GEO 监测" | ✅ ⭐⭐⭐ 真做的话是仓库识别出的最有结构性价值的方向（元假说 A、C） |
| "AI 答案不可被监测" | ❌ 可监测，但难复测 + 难对齐 |

## 七、不做的事

- 不做"如何做反 GEO 监测产品"的执行手册——本仓库立场是研究而非创业指导
- 不做"反 GEO 监测公司估值"分析——无可比数据
- 不做与具体竞品（如海纳 / 蜂鸟 等舆情监测厂商）的对比评测——评测属于工具评测占位段
- 不做"我建议你创业做反 GEO 监测"的建议——研究指出空白，但不下创业建议
- 不做"反 GEO 监测能 100% 解决幻觉" 的承诺式推介——本卡多次强调"监测 ≠ 干预"
- 不评判 LLM 平台的内容审核能力——平台不预先评判优劣是仓库基调

## 八、待解决的未知 / 公开数据空白

1. **上市公司公关 / 法务部门的 AI 监测预算占舆情总预算的比例**——无独立调研
2. **AI 幻觉率在不同行业 / 主体类型的差异**——无系统数据；E2 实验的目标
3. **LLM 厂商是否会自建"品牌方监测后台"**——元假说 G 的核心数据空白
4. **跨 LLM 个性化对监测的影响程度**——不同账号同 query 答案差异有多大，缺方法论
5. **监管细则时间表**——《生成式人工智能服务管理办法》后续配套细则的发布节奏
6. **品牌方对"AI 答案中的负面"的耐受度**（vs 传统媒体负面）——无量化数据
7. **当前已存在的"伪反 GEO 监测"产品的真实覆盖度**——多数所谓"AI 舆情监测"产品的能力上限不公开
8. **法务驱动的反 GEO 监测在何种案例下首次被监管 / 司法采纳为证据**——纵向监测目标
