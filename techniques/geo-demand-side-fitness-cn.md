# GEO 需求侧场景适配地图

> 🔄 **2026-05-04 审计修订标注**（[`audits/2026-05-04_meta-hypothesis-audit.md`](../audits/2026-05-04_meta-hypothesis-audit.md)）：本卡 5 条元假说判决 **A ✓ / B ◐ / C ✗ / D ◐ / E ✓**——核心错位是本卡把 GEO 需求等同于"用户消费决策路径"，遗漏**防御 + 品牌可见度 + 合规留痕**这一维度（央企 / 上市公司大额采购的实际动机）。原文保留，修订标注见每条假说下方 🔄 段。
>
> ⚠️ **状态：concept**（本卡是仓库 14+1 张卡里**第一张需求侧卡**——前 14 张全部讨论"信号 → LLM 引用"，本卡讨论这条链条**前置的一个问题**：用户是不是真的在用 LLM 做这类决策？）
>
> 本卡做四件事：
>
> 1. 拆开"GEO 需求"为三层（用户层 / 场景层 / 决策路径位置层），避免"AI 用户超 10 亿"这类总量话术混淆 GEO 真实可寻址市场
> 2. 给出**用户类型 × 问题类型**矩阵，识别 GEO 真正有效的窄场景（4 类）和无效场景（多数）
> 3. 拆"用 LLM" vs "用 LLM 做决策"——前者是渗透率，后者才是 GEO 价值基础
> 4. 给出**与 GEO 服务商客户画像话术的对照**——多数厂商 6-7 类客户画像里只有 2-3 类是真买家
>
> ⚠️ **怀疑锚点 1**：[C-SEO Bench](https://openreview.net/forum?id=oTeixD3oZO) 严格控制下多数 GEO 方法基本无效。本卡更进一步：**多数 GEO 推广目标场景的"无效"不在信号侧，而在需求侧**——用户从未在该场景使用 LLM 做决策，再多信号优化都无意义
>
> ⚠️ **怀疑锚点 2**："AI 用户超 10 亿 / 月活 3.2 亿"等数字常被服务商引为 GEO 必投依据，但**月活 ≠ 决策路径渗透率**——LLM 在中国消费决策路径中的位置远不如英文场景。中文用户消费决策主路径仍是小红书 → 淘宝评论 → 微信群朋友推荐，LLM 是补充节点
>
> ⚠️ **怀疑锚点 3**：本卡的矩阵基于行业观察 + 仓库其他卡的元假说衍生，**无大样本独立调研支撑**——量化精度低，结构判断价值高。读者应将本卡当**决策清单**而非**事实陈述**

## 一、概念拆解：什么是"GEO 需求"

把"需求"拆开三层，避免单一指标话术混淆：

### L1 用户层

- 谁在用 LLM？
- 衡量：MAU / DAU / 渗透率（按年龄 / 职业 / 地区切片）
- **常被服务商引用**：3.2 亿 MAU / 10 亿用户经 AI 获取信息

### L2 场景层

- 用户在 LLM 上做什么类型的事？
- 衡量：query 类型分布、停留时长、复用频次
- **常被服务商混淆**：把"翻译 / 写作辅助" 用量算入"消费决策" 用量

### L3 决策路径位置层

- LLM 在该决策中处于什么位置？
- 三档：
  - **首选节点**：用户先打开 LLM 输入需求，根据答案直接行动
  - **补充节点**：用户在传统平台（小红书 / 淘宝 / 抖音 / 朋友推荐）形成初步判断后，用 LLM 做"二次验证 / 反向求证"
  - **边缘 / 无关节点**：用户决策完全不经过 LLM
- **GEO 真正的市场基础**：场景中 LLM 处于"首选节点"或"高频补充节点"——其他位置 GEO 优化效果天花板极低

### 为什么这三层必须分开

服务商话术的常见伪命题：

> "10 亿用户用 AI 获取信息（L1） → 你的品牌应该被 AI 看见（L3） → 所以你应该投 GEO"

这个推理跳过了 L2 场景层的拆分。L1 的"AI 用户" 大部分是把 LLM 用作**翻译 / 写作 / 算账 / 写代码**——这些场景下 GEO 没有商业价值（没人推广翻译里的品牌）。真正能转化为 GEO 价值的是 L2 中**"涉及品牌 / 供应商 / 产品选择"** 的子场景，且 L3 中 LLM 至少是"高频补充节点"。

## 二、横向对照表

### 2.1 用户类型轴 × 渗透度

| 用户层 | 用 LLM 找答案概率 | 占总人口估计 | 主要用途 | GEO 商业相关性 |
|-------|-----------------|------------|---------|--------------|
| 极客 / IT 从业者 | 高 | < 5% | 工作 / 编程 / 技术调研 | 中（B2B 工具选型时是真买家） |
| 白领 / 知识工作者 | 中-高 | ~20% | 工作辅助为主，**消费决策仍走小红书 / 淘宝** | 中（B2B 选型有效，B2C 消费几无影响） |
| 学生（中学到研究生） | 高 | ~10% | 学习 / 写作业 / 论文 | 低（消费力弱） |
| 新中产消费者 | 低-中 | ~15% | LLM 是补充而非首选 | 中-低 |
| 银发 / 下沉市场 | 极低 | ~50% | 微信 / 抖音 / 百度 | **极低** |

⚠️ 占比为**作者估计**，缺乏中文场景独立公开调研——见第 IX 节"待解决的未知"。

### 2.2 问题类型轴 × LLM 渗透度 × GEO 商业价值

| 问题类型 | LLM 渗透度 | 用户首选渠道 | GEO 商业价值 |
|---------|-----------|------------|------------|
| 高频低决策（翻译 / 算账 / 写邮件） | 极高 | LLM | **零**——没人在翻译里推广品牌 |
| 学习 / 研究类 | 高 | LLM + 知乎 + 百度 | **ToB 学术工具有价值** |
| **B2B 选型** | **高** | LLM + 行业社区 + 销售人脉 | **⭐ 真正有效**——决策周期长，用户愿意查对比 |
| **YMYL 初筛**（健康 / 法律 / 财务） | 中-高 | LLM + 三甲医院 / 律所官网 + 知乎 | **⭐ 部分有效**——用户警惕但当初筛入口 |
| **新概念 / 长尾决策** | 中-高 | LLM | **⭐ LLM 独占**——传统搜索没关键词 |
| **跨语言 / 跨文化** | 高 | LLM | **⭐ 出海双向缺供给** |
| 高客单消费（买房 / 车 / 保险） | 中 | 多渠道交叉验证 | **低**——LLM 不是关键节点 |
| 本地服务（餐厅 / 医院 / 律师） | **低** | 美团 / 大众点评 / 高德 / 微信 | **极低**——LLM 没 LBS / UGC 闭环 |
| 冲动消费（美妆 / 零食 / 衣服） | **极低** | 小红书 / 抖音 / 淘宝 | **零** |

**结论**：⭐ 标记的 4 类是 GEO 真正有商业空间的窄场景。其他场景下，GEO 优化的天花板由**用户根本不来 LLM** 决定，再强的信号优化也无法破局。

### 2.3 决策路径位置 × 行业差异

按 L3 决策路径位置归类各行业：

| LLM 位置 | 行业例 | GEO 投入合理性 |
|---------|-------|--------------|
| **首选节点** | B2B SaaS 选型、学术工具评估、技术方案对比、出海陪跑 | ⭐⭐⭐ 高——目标客户决策始于 LLM |
| **高频补充节点** | YMYL 初筛、长尾消费品、新概念 / 新品类 | ⭐⭐ 中——LLM 影响首屏印象 |
| **低频补充节点** | 大宗消费（车 / 房 / 保险）、品牌公关验证 | ⭐ 低——LLM 是众多渠道之一 |
| **边缘节点** | B2C 快消、跨境电商终端用户、本地餐饮 | **不应做 GEO**——投入回报结构性低 |
| **无关节点** | 冲动型消费、银发市场、地推依赖业务 | **不可做 GEO**——目标用户压根不在 LLM |

## 三、元假说集

### 元假说 A（强假说）—— 中文消费决策路径中 LLM 是补充节点而非主节点

中文场景下，**绝大多数消费决策的主路径仍是"小红书 / 淘宝评论 / 抖音 / 朋友推荐 → 决策"**，LLM 在路径中的位置以"二次验证 / 反向求证"为主。

支持证据：
- [`citation-attribution-styles-cn.md`](./citation-attribution-styles-cn.md) 元假说 A：中文 LLM 引用源 CTR < 1%——若 LLM 是主节点，引用源 CTR 应远高于此
- 中欧 2026 GEO 白皮书：Google AI Overviews 引用源 CTR 1-1.3%（英文场景）
- 行业观察：豆包 / 元宝 / 千问 等中文 LLM 答案体里的"商品 / 服务推荐"位次几乎不可点击

强反话术：
- ❌ "62.2% 用户依赖 AI 答案做决策"
- ❌ "AI 时代用户先问 AI 再下单"

可证伪性：F——可通过用户行为日志或访谈量化"先 LLM / 后 LLM / 不用 LLM"在不同品类的占比。

### 元假说 B（强假说）—— GEO 真正有效的场景只有 4 类

按 L3 决策路径位置筛，**仅 B2B 选型 / YMYL 初筛 / 长尾决策 / 跨语言** 四类满足"LLM 至少为高频补充节点"。

强反话术：
- ❌ "B2C 电商应做 GEO"——用户路径在小红书 / 淘宝 / 抖音
- ❌ "跨境出海应做中文 GEO"——目标用户在 TikTok / Google / Amazon，不在中文 LLM
- ❌ "本地服务应做 GEO"——LLM 没有 LBS / UGC 数据闭环

可证伪性：P——需先操作化定义"有效"为可观测谓词（例如 LLM 引导的转化占该品类总转化的 ≥ 5%），再分品类测量。

> 🔄 **2026-05-04 审计修订**（判决 ◐，遗漏第 5 类）：F4（百分点国标 + 党校案例）+ F5（央企招标：天翼云 137 万 / 东风 Honda 1 年 6 平台 / 海信 / 东阿阿胶 / 立白 / 招行 / 华为 / 腾讯 / 国家电网）+ F6（信通院起草、AIIA 安全承诺）显示**第 5 类窄场景**被遗漏：**品牌防御 + 合规留痕 + AI 幻觉监测**。
>
> 这一类客户群体是央企 + 上市公司，决策人是公关 / 法务（与 [`anti-geo-monitoring-cn.md`](./anti-geo-monitoring-cn.md) 元假说 B 一致）。原 4 类窄场景应升级为 5 类：**B2B 选型 / YMYL 初筛 / 长尾决策 / 跨语言 / 品牌防御 + 合规**。

### 元假说 C（强反话术，⭐⭐⭐）—— 服务商客户画像越宽，越说明销售在勉强匹配市场

代表样本：探词科技手册 6 类客户画像里 **4 类（B2C 电商 / 跨境出海 / 品牌公关 / 中小微）几乎全部踩在 GEO 无效区**——客户画像越宽，越说明厂商不能精确识别真买家，必须用宽口径触达填补销售漏斗。

强反话术：
- ❌ "GEO 适用所有行业"——本卡 2.2 节直接证伪
- ❌ "GEO 是所有企业的必修课"——多数行业目标用户不在 LLM

可证伪性：F——可通过收集 ≥ 3 家 GEO 服务商客户列表，分析行业分布与 4 类有效场景的重合度。

> 🔄 **2026-05-04 审计修订**（判决 ✗，逻辑错位）：F3 智推时代客户（三七互娱 / 作业帮 / 小鹏 / 理想）+ F5 飞柚 GEO 服务（招行 / 华为 / 腾讯 / 国家电网）+ F1 迈富时（世界 500 强 80+）跨多行业但都是大客户在真付钱。"画像宽 = 销售勉强匹配"的逻辑只在**尾部草莽期玩家**成立，对头部 / 腰部不成立——大客户本身就横跨多行业。
>
> 修订方向：本假说应**降级**为"针对尾部 GEO 公司"的诊断启发，**不能作为整体反话术**——头部 / 腰部公司的宽客户画像可能恰恰反映其大客户密度，与销售勉强匹配无关。基于探词单样本→行业基线的外推被证伪。

### 元假说 D —— 三类行业在 LLM 上几乎不可能被影响

本地服务 / 冲动消费 / 大宗高客单，三类行业的传统决策路径具有**结构性壁垒**：

- 本地服务：LBS / UGC 评分 / 即时性是 LLM 没有的能力
- 冲动消费：决策时长短（< 1 分钟），用户没有打开 LLM 的耐心
- 大宗高客单：决策周期长但每次决策点都依赖**线下试驾 / 看房 / 顾问咨询**，LLM 不在关键节点

强反话术：
- ❌ "汽车品牌应做 GEO"
- ❌ "美妆 / 时尚品牌应做 GEO"
- ❌ "餐厅应做 GEO"

可证伪性：F——分行业可独立测量。

> 🔄 **2026-05-04 审计修订**（判决 ◐，混淆消费路径与采购动机）：F5 东风 Honda（汽车 = 大宗）招标 6 个 AI 平台车型推荐 1 年 + 东阿阿胶（健康消费）+ 立白（日化）+ 海信集团均为大宗 / 冲动 / 本地相关行业，但都在投钱做 GEO。
>
> 修订方向：消费路径维度仍成立——**用户**确实不会通过 LLM 决策本地服务 / 冲动消费 / 大宗高客单。但这三类行业的**品牌方**仍有 GEO 采购动机：**防御性可见度** + **AI 幻觉监测**（LLM 编造的产品信息可能引发投诉 / 监管）。本假说应区分"用户路径"vs"品牌采购动机"——两个维度独立。

### 元假说 E（强假说）—— "用 LLM" vs "通过 LLM 做决策"是两个数量级不同的市场

L1 用户层（用 LLM 的人）和 L2/L3 GEO 价值场景（在 LLM 上做品牌 / 供应商决策的人）之间存在**至少一个数量级的差距**。

支持证据：
- 多数 LLM 用户用 LLM 做翻译 / 写作 / 编程 / 学习辅助——L2 中"消费决策" 类 query 占比 < 10%（行业观察 + 小样本日志）
- L3 决策路径"首选节点"占比 < L2 中"消费决策"类 query 的 30%（多数仍是补充节点）

推论：3.2 亿 LLM MAU × 10% × 30% ≈ 900 万左右是真正"在 LLM 上做品牌 / 供应商决策"的用户——这才是 GEO 真实可寻址市场，而非 3.2 亿。

可证伪性：F——可通过分层抽样调研直接量化。

## 四、3 个实验设计建议

### E1 ⭐⭐⭐ 中文消费者真实决策路径分品类调研

**目的**：量化"先 LLM / 后 LLM / 不用 LLM" 在不同品类的占比，直接证伪 / 支持元假说 A、E。

**方法**：
- 招募 ≥ 500 名 18-55 岁城市网民，分层（按职业 + 年龄 + 地区）
- 列出 ≥ 12 类消费场景（覆盖本卡 2.2 节问题类型轴）
- 每类场景问：上次该品类决策时，你是否用了 LLM？在什么阶段？是首选还是验证？
- 输出：分品类的"LLM 在路径中位置"分布图

**预算**：第三方调研公司报价 ¥10-30 万；自建样本可降到 ¥3-5 万

**预期产出**：
- 直接证伪 / 支持元假说 A
- 给元假说 B 提供品类级证据
- 给所有 GEO ROI 决策提供需求侧基准

### E2 ⭐⭐ B2B 选型场景 LLM 渗透率独立测量

**目的**：在已识别为"GEO 真有效"的 B2B 选型场景内，量化 LLM 实际被使用的比例。

**方法**：
- 招募 ≥ 100 名 B2B 决策者（采购经理 / 技术负责人 / 市场总监）
- 跟踪近 3 个月内 ≥ 1 次的供应商选型经历
- 记录信息源接触顺序、LLM 使用环节、最终决策依据

**预算**：¥3-8 万

**预期产出**：
- 元假说 B 的 B2B 子场景实证
- 为 [`geo-roi-framework-cn.md`](./geo-roi-framework-cn.md) B2B SaaS 类目提供 ROI 输入参数

### 不建议做的实验

- ❌ **泛"AI 用户量"调研**——已有 QuestMobile / 艾瑞等大量第三方报告，且与 GEO 商业价值无强因果
- ❌ **跨平台 LLM MAU 对比**——平台官方数据已部分公开，调研边际产出低
- ❌ **"用户对 AI 答案的信任度" 抽象问卷**——抽象问题答案波动大，转化为 GEO 决策依据的链路太长

## 五、与 GEO 服务商话术的对照

| 服务商话术 | 本卡判定 |
|----------|---------|
| "AI 用户超 10 亿，AI 流量红利" | ⚠️ 真，但 L1 ≠ L2 ≠ L3，与 GEO 价值无强因果（元假说 E） |
| "62.2% 用户依赖 AI 答案做决策" | ⚠️ 需拆是哪类决策——多数是工作辅助 / 学习类（非 GEO 价值场景） |

## 六、与本仓库其他卡片的关系

本卡是仓库**第一张需求侧卡**，前置于现有所有信号侧讨论：

- **前置于 [`dimension-map-cn.md`](./dimension-map-cn.md) 全部 8 维度**——8 维度全部假设需求存在；本卡讨论需求是否存在
- **互补 [`geo-roi-framework-cn.md`](./geo-roi-framework-cn.md)**——后者讨论"做的成本与回报"，本卡讨论"是否值得开始判断成本与回报"
- **配套 [`overseas-cn-deployment.md`](./overseas-cn-deployment.md)**——出海双向（中→外 / 外→中）都是 GEO 真有效区的子集
- **支持 [`citation-attribution-styles-cn.md`](./citation-attribution-styles-cn.md) 元假说 A**——CTR < 1% 暗示 LLM 不是流量入口，与本卡元假说 A 相互印证
- **限定 [`prompt-level-seo-cn.md`](./prompt-level-seo-cn.md) 适用范围**——prompt-level 优化只在用户真用 LLM 决策的场景里有意义

衍生回填到 [`dimension-map-cn.md`](./dimension-map-cn.md) 的建议：把 8 维度更新为 **D0（需求侧前置维度）+ D1-D8（信号侧维度）** 的结构——具体回填见衍生 BACKLOG。

## 七、不做的事

- 不做"AI 渗透率综合调研"——不是本仓库职责，已有第三方机构覆盖
- 不做"该不该做 GEO"的二元决策建议——本卡只识别需求场景，决策需结合 [`geo-roi-framework-cn.md`](./geo-roi-framework-cn.md) 的成本侧
- 不预测 LLM 在 5-10 年后的位置——只做当前快照
- 不评判 LLM 平台优劣——平台不预先评判是仓库基调
- 不做"如何劝服客户做 GEO" 的销售话术——这反过来变成本仓库要拆解的对象
- 不在本卡内生成具体的"LLM 决策路径占比"数字——元假说 A、E 给的占比是结构判断估计，需第三方调研确证

## 八、待解决的未知 / 公开数据空白

1. **中文消费者 LLM 渗透率按品类的公开数据稀缺**——QuestMobile / 艾瑞做了总量数据，但**按品类 × 决策阶段** 的细粒度数据空白
2. **"决策路径中 LLM 位置"的量化方法学不成熟**——访谈 / 行为日志 / 自我报告各有偏差，缺方法论标准
3. **YMYL 类 LLM 信任度的纵向数据缺失**——用户警惕度可能在波动，没有连续跟踪
4. **跨年龄 / 跨地区的渗透率差异**——中文场景没有 Pew Research 类机构系统跟踪
5. **B2B 决策路径中 LLM 位置的独立调研**——Gartner / Forrester 类机构尚未发布中文场景报告
6. **"用 LLM" vs "通过 LLM 做决策" 的数量级差距**（元假说 E）—— 行业观察 + 估计，无大样本独立调研
7. **不同 LLM 平台的"决策类 query 占比"**——豆包 / 元宝 / 千问 / 文心官方未披露
8. **中文 LLM 的"零点击"率数据**——服务商话术常用，但来源未披露
