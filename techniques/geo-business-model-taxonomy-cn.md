# 中文 GEO 商业模式八层分类

> 🔄 **2026-05-04 审计修订标注**（[`audits/2026-05-04_meta-hypothesis-audit.md`](../audits/2026-05-04_meta-hypothesis-audit.md)）：本卡 6 条元假说中 **A ✗ / B ◐ / F ◐**，C/D/E 被反转事实强化。原文保留——具体修订标注见每条假说下方 🔄 段。
>
> ⚠️ **状态：concept**（基于探词科技手册拆解 + 仓库 14+1 张技术卡的元假说收敛 + SEO 行业类比观察。**所有具体公司分类基于公开物料**，未做厂商访谈）
>
> 本卡是仓库**供给侧**的第一张系统卡，与 [`geo-demand-side-fitness-cn.md`](./geo-demand-side-fitness-cn.md)（需求侧）、[`geo-roi-framework-cn.md`](./geo-roi-framework-cn.md)（决策侧）形成**需求 + 信号 + 模式 + ROI**四轴完整研究地图。
>
> 本卡做四件事：
>
> 1. 把"GEO 业务"按供给侧拆为八层（数据 / 工具 / 内容 / 服务 / 平台 / 防御 / 标准 / **横向扩展**），每层独立评估——h 层"横向扩展"在 2026-05-04 审计后新增，承载迈富时 / 百分点 / 蓝色光标系等"上市营销云 + 数据公司加 GEO 模块"画像
> 2. 给出每层的**已存在玩家 / 主要客户 / 关键风险 / 被平台原生功能取代的风险**
> 3. 拆"商业模式合理性"≠ "客户价值"——某些模式即使无效也能赚钱（培训认证型），某些模式即使有效也难做大（战略咨询型）
> 4. 与 GEO 服务商常见自我定位话术对照（"我们做的是 SaaS"、"OEM 模式扩张性强"等）
>
> ⚠️ **怀疑锚点 1**：[C-SEO Bench](https://openreview.net/forum?id=oTeixD3oZO) 在严格控制下多数 GEO 方法基本无效——这意味着**多数 GEO 商业模式建立在效果不稳定的供给上**，模式合理性需结合效果稳定性独立评估，不能因为"有人付费"就推断"模式合理"
>
> ⚠️ **怀疑锚点 2**：**LLM 平台一旦自建广告位**（豆包 / 元宝 / 千问任意一家做 LLM 时代竞价排名），中介型 GEO 模式（c/d 层）整体被腰斩——这是悬在所有 GEO 公司头上的结构性风险，本卡多次回到这一点
>
> ⚠️ **怀疑锚点 3**："GEO 是赛道 / 万亿市场"叙事本身是**商业模式包装**而非客观市场判断。本卡按"是否能在平台原生功能压力下持续存在" 评估每层，不是按"市场叙事大小"评估

## 一、概念澄清

### 1.1 商业模式 ≠ 客户价值

两个独立维度：

- **客户价值**：客户买了之后是否真获得效果（仓库 14 张信号侧卡 + 需求侧卡讨论的对象）
- **商业模式合理性**：供给方能否长期赚钱、能否扩张、能否抵御平台风险

四象限：

|  | 客户价值高 | 客户价值低 |
|---|----------|----------|
| **模式合理** | 长期赢家（罕见） | "合法暴利"——培训 / 认证 / 焦虑变现 |
| **模式不合理** | 难做大——战略咨询 / 高度定制 | 双重无效——多数 GEO 短命公司 |

仓库立场：客户价值由仓库 14+1 张技术卡评估；本卡只评估**模式合理性**——两件事不应混为一谈。

### 1.2 三种计费方式 → 三种模式锚点

- **按效果计费**：客户付费与"被 LLM 引用 / 位次提升" 挂钩——产品形态是 RaaS（探词的主力）
- **按服务计费**：固定费用买工具 / 报告 / 咨询——产品形态是 SaaS / 咨询
- **按规则计费**：通过制定标准 / 认证 / 牌照获利——产品形态是协议 / 培训 / 行业组织

三种锚点对应不同的扩张曲线和风险结构，下面八层按这三种锚点分布。

### 1.3 区分"GEO 价值链"vs "GEO 商业模式"

GEO 价值链 = 客户从"决定做 GEO" 到"看到结果" 的工序流：诊断 → 内容生产 → 信源铺设 → 监测 → 复盘。GEO 商业模式 = 在这条价值链上**选哪一段切入 + 用什么计费方式**。

多数厂商**全链路打包**（探词的四模块就是全链路），但**风险也全链路承担**——任何一段崩盘都影响交付。

打 ⭐⭐⭐ 的方向：**a 第三方指数 + b 垂直工具 + d 战略咨询和场景陪跑 + f 反 GEO 监测 + g 标准推动 + h 横向扩展（迈富时 / 百分点路径）**。

### 2.2 八层 × 投入产出结构

| 层 | 资金强度 | 人力强度 | 时间强度 | 客户单价 | 利润率 |
|---|--------|---------|---------|--------|-------|
| a 数据 / 监测 | 中（爬虫 + 存储） | 低（一旦自动化） | 长（建数据资产） | 中 | 高 |
| b 工具 / SaaS | 低-中 | 低 | 中 | 低-中 | 高 |
| c 内容 / 信源 | 中（采购媒体） | 中（写作 + 审核） | 短 | 低 | **低**（议价权在媒体） |
| d 服务 / 咨询 | 低 | **高**（重人工） | 短 | 高 | **低-中**（人力天花板） |
| e 平台 / 生态 | 中 | 中 | 长（网络效应） | 看规模 | 中-高（一旦起来） |
| f 反向 / 防御 | 中 | 中 | 中 | 高（公关 + 法务预算大） | 高 |
| g 标准 / 协议 | 低（影响力为主） | 低 | **极长** | 间接 | **极高**（一旦定下来） |
| h 横向扩展 | **高**（上市资源 / 既有渠道复用） | 低-中（团队复用） | 短（既有交付能力） | **高**（大客户基础） | 中-高 |

c 层和 d 层是**当前中国 GEO 行业最拥挤的两层**，但也是利润率最低、平台风险最高的——这是探词们 / 创业 SaaS 玩家的处境。**h 层（迈富时 / 百分点）是另一个完全不同的画像**——既有客户密度 + 上市资源 + 合规背书，进入门槛对创业公司是结构性的，但对腰部上市公司是顺势扩展。

### 2.3 八层 × 平台原生功能取代风险

把"LLM 平台自建什么功能就可以替代该层" 列出来：

| 层 | 平台一旦自建什么 → 该层被取代 |
|---|---------------------------|
| a 数据 / 监测 | 平台开放官方 API 给客户查"我的品牌在我家答案里出现了多少次"——中等取代风险，但因为客户需要**跨平台对比**，第三方独立性仍有空间 |
| b 工具 / SaaS | 平台原生提供"内容 LLM 友好度检查"工具——中等取代风险 |
| c 内容 / 信源 | 平台开放"商家直接上传内容到 LLM 知识库"通道（类似豆包知识库 / 千问百炼）——**高取代风险** |
| d 服务 / 咨询 | 平台上线"AI 竞价排名"（最致命）—— **完全取代** |
| e 平台 / 生态 | 部分自建（OEM 被同行替代，市场被生态替代） |
| f 反向 / 防御 | 平台不会自建"帮你监测竞品"——**几乎无平台风险** |
| g 标准 / 协议 | 平台是**协议被定义的对象**，不是制定者——**几乎无平台风险** |
| h 横向扩展 | 平台变现冲击 GEO 模块时，营销云 / 数据公司本体仍在——**平台风险被稀释**（GEO 是上市公司多产品矩阵的一部分，不是全部） |

**结论**：f 反向 / 防御 + g 标准 / 协议 + h 横向扩展 是**平台风险最低的三层**——f / g 是中文场景空白，h 是上市公司既有玩家已占。仓库立场下：创业公司的结构性最优方向是 a 第三方指数 + b 垂直工具 + d 战略咨询 / 场景陪跑 + f 反 GEO 监测 + g 标准推动；上市公司的结构性最优方向是 h（迈富时 / 百分点路径）。

## 三、元假说集

> 🔄 **2026-05-04 审计后说明**：h 层"横向扩展"是审计新增的一层，但**未独立成新元假说**——h 层的判断已分散落入 B / F 两条假说的修订段（B 修订指出迈富时证明 RaaS 上市规模可达；F 修订指出垂直陪跑 + 横向大客户密度都是稳的位置）。如果未来需要独立成 h 层专题元假说，候选命题是"上市营销云 + 数据公司加 GEO 模块的客户保有率显著高于纯创业 GEO SaaS"——这是 [`tech-cn-geo-industry-cr5-tracker`](../BACKLOG.md) BACKLOG 项的可证伪假设源头。

### 元假说 A（强反话术，⭐⭐⭐）—— LLM 平台一旦自建广告位，c/d 层中介模式整体被腰斩

LLM 平台（豆包 / 元宝 / 千问 / 文心）任意一家上线"AI 时代竞价排名"，即让品牌方直接付费给平台，答案体里出现广告位/赞助商位次，**c/d 层（内容铺量 + 效果代运营）的中介定价权立刻消失**。这与传统搜索引擎广告（Google AdWords / 百度推广）取代 SEO 中介的剧本一致。

支持证据：
- 字节早已在抖音 / 头条端做信息流广告，技术 + 销售团队完全具备做 LLM 广告位的能力
- 千问与淘宝深度绑定，电商类广告天然有商业模式参考
- 元宝可对接微信广告系统
- [`platform-search-blockade-cn.md`](./platform-search-blockade-cn.md) 揭示的平台行为模式：**护城河变现优先，开放性次之**

强反话术：
- ❌ "GEO 是 AI 时代的长期赛道"——多数中介模式的赛道生命周期由平台决策决定，不超过 3-5 年
- ❌ "AI 时代去中介化，GEO 公司机会大"——恰恰相反，AI 时代平台话语权增强，中介更被动

可证伪性：F——直接监测豆包 / 元宝 / 千问 / 文心 是否上线"广告位 / 赞助商位 / 品牌优先" 等功能即可验证；当前阶段 = 待发生事件，可设监测告警。

> 🔄 **2026-05-04 审计修订**（判决 ✗，方向对形态错）：F7 反转事实显示平台变现路径**不是凤巢式竞价**，是**订阅 + 电商佣金**混合（智推时代创始人 2025/11 原话："国内只有 GEO，平台没开放商业化"；豆包做付费订阅 + 链接抖音商店；阿里 5 年云+AI 1000 亿美金混合路径）。
>
> "平台变现冲击 c/d 层中介模式" 这一**方向**仍成立，但**单押"广告位"**为唯一风险形态被证伪。实际威胁形态有 4 种：
>
> 1. **订阅制**：消费侧用户付费给平台，c/d 层服务商被绕过
> 2. **电商佣金**：平台直接抽供给侧成交（豆包 → 抖音商店、千问 → 淘宝），c/d 层被绕过
> 3. **自建品牌方监测后台**（已在 [`anti-geo-monitoring-cn.md`](./anti-geo-monitoring-cn.md) 元假说 G 部分覆盖）
> 4. **API 收费**：B2B 端开发者直接接入，绕过中间服务商
>
> 强反话术应改为："GEO 是 AI 时代长期赛道" → "c/d 层中介模式生命周期由平台变现路径决定，4 种形态任一上线即冲击"。
>

#### 元假说 A（升级版，2026-05-04） —— 平台变现冲击 c/d 层：4 种形态，任一上线即冲击中介定价权

LLM 平台（豆包 / 元宝 / 千问 / 文心 / Kimi）的**变现机制成熟**会冲击 c/d 层（内容铺量 + 效果代运营）的中介定价权。冲击形态不是单一的"凤巢式广告位"，而是 4 种独立但叠加的路径——**任一形态上线，都让 c/d 层的部分价值被平台直接抽走**：

##### 形态 1：订阅制（消费侧绕过）

**机制**：消费侧用户付费给平台（豆包付费版 / 文心一言会员 / Kimi 付费版），平台从用户口袋直接收钱。

**对 c/d 层的冲击**：
- c 层（内容铺量）：付费用户对答案质量要求更高 → 平台对训练 / RAG 数据源筛选更严 → 低质 SEO 内容被过滤的概率更高
- d 层（效果代运营）：客户的 ROI 计算分子（被引用次数）变小 + 效果不可控性变大

**当前已有信号**：
- 豆包付费订阅已上线（按月 / 按年套餐）
- 文心一言会员定价 49.9-59.9 元 / 月（2025 年）
- Kimi 探索版按 query 计费（先锋玩家）

**可证伪性**：F——直接观察各平台订阅版的用户数披露 + 答案质量是否较免费版有显著差异。

##### 形态 2：电商佣金（供给侧绕过）

**机制**：平台直接抽**供给侧**成交佣金（豆包 → 抖音商店、千问 → 淘宝、元宝 → 微信小程序 + 视频号小店），把答案体直接连到电商通路，跳过 c/d 层的内容铺量。

**对 c/d 层的冲击**：
- c 层（内容铺量）：消费类品牌不再需要在第三方铺内容"碰运气被引用"——直接把店铺挂到平台电商通路即可获客
- d 层（效果代运营）：消费品类的 RaaS 模式失去存在意义——客户为什么花钱给中介买"被引用率"？直接给平台抽佣就行

**当前已有信号**：
- 豆包 2025/Q4 已链接抖音商店
- 千问与淘宝绑定（"AI 下单"路径）
- ChatGPT Atlas 跳蓝链 → Shopify
- 元宝可路由到微信小程序 + 视频号小店

**对 c/d 层的实际伤害**：电商佣金对**消费类**品牌（B2C / 美妆 / 零食 / 服装 / 家居）冲击最大；对 B2B 选型 / YMYL / 政企采购影响小（平台没有这些场景的电商通路）——**c/d 层的剩余价值集中在非电商可路由场景**。

**可证伪性**：F——监测各平台答案体中"商品卡 / 跳转店铺"的出现频率 + 抽佣率公开数据。

##### 形态 3：自建品牌方监测后台

**机制**：LLM 平台上线"品牌方查询自家在 LLM 中表现"的官方后台（类比 Google Search Console for SEO）。一旦上线，**a 层（数据 / 监测）的部分价值**被平台直接收回。

**对 c/d 层的冲击**：
- a 层（数据 / 监测）：单平台监测的部分场景（N1 品牌可见度）被官方后台部分替代——但跨平台监测 + 第三方独立性仍有空间（详见 [`anti-geo-monitoring-cn.md`](./anti-geo-monitoring-cn.md) 元假说 G）
- d 层（效果代运营）：客户的"看效果"环节被平台官方后台标准化，d 层的 ROI 评估话术失效——客户可以直接看官方数据

**当前已有信号**：
- Google Search Console 是成熟先例（搜索引擎对 SEO 中介的官方对位工具）
- 中文 LLM 平台暂无对应后台
- 但平台已具备技术能力（豆包 / 千问 / 元宝 / 文心都有 B 端开发者后台）

**对 a 层的限制**：a 层不会被完全取代——客户仍需要**跨平台对比** + **独立第三方报告**作为合规留痕（参考 SEO 工具行业 20 年历史：Ahrefs / Semrush 在 Search Console 上线后仍活跃）。

**可证伪性**：F——直接监测各平台官方"开发者中心 / 商家中心 / 品牌中心" 是否上线"我家在 AI 答案里的可见度查询" 模块。

##### 形态 4：API 收费（B2B 端绕过）

**机制**：平台对 B2B 端开发者直接收 API 费（按 token / 按调用），让企业客户**自己接入 LLM 能力**而不是通过中间服务商。

**对 c/d 层的冲击**：
- c 层（内容铺量）：企业客户接入 API 后，可能自建私有 RAG / 企业知识库，绕过对外公开内容铺量
- d 层（效果代运营）：企业客户直接对接 API，d 层的效果代运营失去对接对象

**当前已有信号**：
- 阿里通义千问 API（百炼平台）按 token 计费
- 字节豆包 API（火山引擎）按 token 计费
- 腾讯混元 API（开发者平台）已商业化
- 阿里 5 年云+AI 商业化目标 1000 亿美金（API 是核心组成）

**对 c/d 层的实际伤害**：API 收费对**B2B SaaS 场景**冲击最大——B2B 客户付费意愿强 + 自有技术团队 + 数据安全要求高，更倾向直接接 API 而不是通过中间服务商。**c/d 层的 B2B 业务量会被 API 商业化挤压**。

**可证伪性**：F——监测各平台 API 公开定价 + 企业客户披露的 API 调用量。

##### 4 形态的叠加效应

4 种形态**不是互斥**，而是叠加压制 c/d 层的不同子市场：

| 形态 | 主要影响 c/d 层的客户类型 | 主要影响层 |
|------|------------------------|----------|
| 订阅制 | 消费侧（付费用户提高质量门槛） | c 层 |
| 电商佣金 | B2C 消费类品牌 | c + d 层 |
| 自建品牌方监测后台 | 大型品牌（防御侧） | a + d 层 |
| API 收费 | B2B 企业客户 | c + d 层 |

**c/d 层在 4 形态全部上线后剩余的价值**：
- **非电商可路由场景**（B2B 选型 / YMYL / 政企采购）的内容铺量
- **跨平台监测 + 独立第三方报告**（不被任何单一平台官方后台覆盖）
- **垂直陪跑 + 行业 know-how**（绑定行业 know-how 而非平台规则——与元假说 F 一致）

**强反话术（升级版，从 2 条扩为 6 条）**：

- ❌ "GEO 是 AI 时代长期赛道"——c/d 层中介模式生命周期由平台变现路径决定，4 种形态任一上线即冲击
- ❌ "AI 时代去中介化，GEO 公司机会大"——AI 时代平台话语权增强，中介更被动
- ❌ "等平台上广告位再说"——4 种形态中**只有形态 3（自建品牌方监测后台）尚未在中文场景上线**；订阅制 + 电商佣金 + API 收费都已部分上线
- ❌ "我们做的不是中介是 SaaS"——SaaS 模式仍受形态 4（API 收费）冲击；多数所谓 SaaS 的真实主收入仍是 RaaS（与元假说 B 一致）
- ❌ "平台不会做监测后台抢自己的客户"——Google Search Console 20 年历史已反驳此命题
- ❌ "API 收费是 B2B 利好"——对**纯 c/d 层中介**是冲击；只有 b 层垂直工具能与 API 形成互补

**可证伪性（升级版）**：F——
- 形态 1：监测各平台订阅版用户数披露 + 答案质量分层
- 形态 2：监测答案体中商品卡出现频率 + 平台抽佣率
- 形态 3：监测各平台是否上线"商家品牌可见度查询"模块
- 形态 4：监测 API 公开定价 + 企业 API 调用量披露

**4 形态分别可设独立监测告警**（衍生为 BACKLOG 项 [`exp-cn-llm-platform-4-monetization-form-monitor`](../BACKLOG.md)）——比原假说"广告位上线"单一告警更全面。

### 元假说 B（强假说）—— 纯执行型代运营毛利天花板由人力成本决定

RaaS 模式（探词 + 跟随者）每单需要人工监测 + 内容生产 + 媒体投放，扩张曲线是线性的（多 1 个客户需多 0.5-1 个交付人手）。在中国 B2B 营销服务市场的标准估值倍数下（PS 1-3 倍、PE 5-15 倍），**RaaS 模式的估值天花板远低于纯 SaaS**。

支持证据：
- SEO 行业成熟期（2010-2015）的代运营公司多数最终被并购或退出，难以独立成上市公司
- 探词团队结构（3 研发 / 3 市场 / 3 交付 / 1 董事长）已是咨询公司配比，扩张需线性增加交付人手

强反话术：
- ❌ "我们做的是 SaaS"（多数所谓 SaaS 实际收入 < 10%，主收入仍是 RaaS）
- ❌ "AI 时代营销 SaaS 是赛道"——多数中文营销 SaaS 实际是"轻 SaaS + 重服务"

可证伪性：P——需厂商披露真实收入构成。可通过收集 ≥ 5 家 GEO 公司公开融资材料 / 招聘信息间接推算。

> 🔄 **2026-05-04 审计修订**（判决 ◐，过度悲观）：F1 迈富时（02556.HK 港股上市，21 万企业客户、世界 500 强 80+、IDC 中国营销云市占第一 7 年、自研 T-GEO™ 五层架构 + Tforce 千亿参数大模型 + RaaS 退款保障）证明 **RaaS 类公司可达港股上市规模**；F3 智推时代被三七互娱（002555）+ 趣睡科技（301336）两家 A 股上市公司投资。
>
> "毛利天花板由人力成本决定"在**纯小作坊**层面成立，但在**SaaS 化 + 大客户密度 + 行业标准化**到位后，天花板由这三个因子共同决定，远高于纯人力天花板。
>
> 修订方向：B 应保留作"尾部 / 中段 RaaS 公司天花板",但应同时给出"头部 RaaS 公司可上市"作上界。

### 元假说 C（强反话术，⭐⭐⭐）—— 反 GEO 监测是被严重低估的对称赛道

任何进攻型行业必长出对应防御型行业。当前所有玩家都在 c/d 层"帮品牌做 GEO"（进攻方），**反 GEO 监测**（帮品牌识别"竞品在用 GEO 攻击我"、"AI 答案中出现虚假信息"、"舆情通过 LLM 放大"）的供给为零。

潜在客户：品牌公关部门 + 法务部门 + 大型企业舆情监测部门——**付费意愿不亚于进攻方**，且预算来源不同（公关 / 法务，而非营销）。

支持证据：
- SEO 时代的对称剧本：负面 SEO 监测、域名抢注监测、品牌词保护服务
- 大型企业（金融 / 医药 / 能源 / 互联网巨头）已在采购舆情监测服务，把"AI 答案中的品牌呈现监测" 加进现有服务包是顺势扩展

强反话术：
- ❌ "GEO 是品牌主动出击的赛道"——一半的市场在防御侧，且无人占
- ❌ "GEO 服务商之间竞争"——真正的对手是其他品牌的 GEO 服务商，监测是必需品

可证伪性：F——可通过访谈 ≥ 10 家上市公司公关 / 法务总监验证需求强度 + 现有支付能力。

> 🔄 **2026-05-04 审计强化**（判决 ✓✓）：F4 百分点参与 40 项国标 + 党校案例 + 工信部白皮书；F6 信通院《GEO 服务可信基本要求》起草中 + AIIA 安全承诺——大幅强化"防御侧需求被严重低估"判断。

### 元假说 D —— 标准 / 协议层是中文 GEO 长期最高 ROI 的位置

第一个推动 llms.txt / AI content provenance / 中文 LLM 训练数据合规等协议的中国主体，会获得**规则定价权**——后来者要付学费。这是 g 层为什么打 ⭐⭐⭐ 的根本原因：投入低（一两个工程师 + 行业关系），产出极高（一旦标准被采纳，所有 GEO 公司绕不过你）。

支持证据：
- 国际场景：W3C / IETF 等组织的早期推动者长期受益
- 中国场景：《生成式人工智能服务管理办法》《互联网信息服务深度合成管理规定》等已落地，**协议 / 标准窗口期开启**
- SEO 行业类比：robots.txt / sitemap.xml 等协议的早期推动者（Google / Bing）长期获利

强反话术：
- ❌ "标准是政府的事"——标准是**先做出来**再被政府采纳，不是政府主动定
- ⚠️ "我们公司太小做不了标准"——首推者通常正是中等规模公司，大厂反而不愿先动

可证伪性：P——需观察 5-10 年长期效果。短期可观察"是否有中国主体在推 llms.txt 中文版"等具体动作。

> 🔄 **2026-05-04 审计强化**（判决 ✓✓）：F4 百分点已是国标制定参与方先例（40 项 + CMMI 5 + ISO20000）；F6 信通院起草中——标准协议层窗口期已开启，"位置最高 ROI"被反转事实强化。

### 元假说 E —— 培训认证型是新瓶装旧酒的合法暴利

参照 SEO 行业"百度推广认证培训"、信息流行业"巨量学认证"等成熟剧本，**GEO 培训认证**几乎注定有人做且赚钱——客户价值争议但**模式合理性极高**：低人力 / 高利润 / 抗平台风险 / 自然续费。

强反话术：
- ❌ "培训没用是焦虑变现"——焦虑变现是模式描述而非对错判断；这个市场会被填满，问题只是被谁填
- ✅（反向使用）"我们的认证是行业标准"——本卡视角下，**第一个做认证的有资格如此说**

可证伪性：F——直接观察 1-3 年内是否出现"GEO 优化师认证"、"GEO 营销师认证"等培训产品 + 报名规模。

> 🔄 **2026-05-04 审计强化**（判决 ✓）：F6 天津商业大学新工科课程已纳入 GEO 案例 + 14 家联合倡议 + 艾瑞研究报告——培训认证路径已开始落地。

### 元假说 F —— 垂直陪跑（B2B SaaS / YMYL）是平台风险下最稳的位置

绑定行业 know-how 而非平台规则——平台规则一变，**懂行业的人仍懂行业**，但只懂平台规则的人就废了。这与 [`geo-demand-side-fitness-cn.md`](./geo-demand-side-fitness-cn.md) 元假说 B（4 类窄场景）相互印证：

- B2B SaaS 选型：行业 know-how = 各 SaaS 类目的关键决策因子 + 各厂商的差异点
- YMYL（医疗 / 法律 / 财务）：行业 know-how = 合规边界 + 专业表达 + 信任建立逻辑
- 出海陪跑：行业 know-how = 内容审查规则 + 跨文化表达
- 长尾消费：行业 know-how = 细分品类的产品力评估

支持证据：
- 探词手册案例中"百奥云"（B2B 育种工具）、"安邦制药"（医药）、"宁夏圆润"（小众消费）这三类**实际复测概率最高**——因为有效区与垂直 know-how 相关
- SEO 行业类比：垂直 SEO（医疗 SEO / 法律 SEO / 跨境 SEO）的存活率高于通用 SEO

可证伪性：P——可通过纵向跟踪垂直陪跑公司 vs 通用 GEO 公司的 3-5 年存活率验证。

> 🔄 **2026-05-04 审计修订**（判决 ◐，过度二元）：F1 迈富时的横向覆盖（21 万企业客户 + 世界 500 强 80+）证明**横向覆盖 + 大客户密度** 也能稳。"垂直陪跑最稳" 修订为"垂直陪跑 + 横向大客户密度都是稳的位置，纯通用 GEO 不稳"。

## 四、3 个实验设计建议

### E1 ⭐⭐⭐ LLM 平台原生广告位上线监控

**目的**：直接验证元假说 A——LLM 平台何时会自建广告位。这是悬在所有 GEO 公司头上的最大单一风险。

**方法**：
- 监测豆包 / 元宝 / 千问 / 文心 / Kimi / 小红书问一问 6 大平台
- 每周自动抓取首屏答案体，关注"是否出现"广告 / 赞助 / 推广 / 优选 / 合作伙伴 等标识
- 同时监测平台官方文档 / 商业产品页面 / 媒体公告
- 输出：6 平台 × 时间线的广告位上线告警表

**预算**：自建监测 ¥0（爬虫 + Sheet）；外采舆情 ¥3-5 万 / 年

**预期产出**：
- 一旦任何平台上线广告位，全行业格局重估
- 即使不上线，也能看到平台向广告化的"试探信号"（小流量灰度 / 接口调整）

### E2 ⭐⭐ 中国 GEO 公司收入构成调研

**目的**：验证元假说 B——多数所谓"SaaS"模式的厂商，真实主收入仍是 RaaS / 项目制。

**方法**：
- 收集 ≥ 5 家中文 GEO 公司的公开物料（融资 PR / 招聘 JD / 案例文章 / 媒体访谈）
- 按"销售岗 vs 交付岗 vs 研发岗 vs 客户成功岗"占比反推业务结构
- 交付岗占比高 → RaaS 主导；研发岗占比高 → SaaS 主导
- 输出：5 家厂商的 RaaS / SaaS / OEM 收入估计 + 真实业务结构推断

**预算**：人力 ≈ 8-15 小时（全公开信息）

**预期产出**：
- 元假说 B 的间接证据
- 给客户选服务商提供"看招聘看真实业务" 的辅助框架
- 给仓库后续话术拆解卡 #2、#3 提供方法学

### 不建议做的实验

- ❌ **泛"GEO 公司估值排名"**——估值数据多数为非公开 / 选择性披露，与商业模式合理性弱相关
- ❌ **"GEO 客户满意度调研"**——满意度受销售关系影响大，数据噪声高
- ❌ **预测某家具体公司命运**——本仓库立场是研究而非商业预测

## 五、与 GEO 服务商话术的对照

| 服务商话术 | 本卡判定 |
|----------|---------|
| "我们做的是 SaaS" | ⚠️ 多数所谓 SaaS 实际收入 < 10%，主收入仍是 RaaS（元假说 B） |
| "我们的 OEM / 源码模式扩张性强" | ❌ 长期会被山寨化稀释（千博系统剧本） |
| "AI 时代营销 SaaS 是赛道" | ⚠️ 多数厂商实际是"轻 SaaS + 重服务" |
| "我们独家覆盖 N+ 平台" | ❌ 平台一改 UI 全部返工，覆盖广 = 技术债指数级 |
| "效果按 query 计费" | ✅ 是**聪明的产品化**——把不可承诺的"业务效果"降维到可验证的"单 query 位次"，承诺难度大幅下降。这是探词手册里少数值得肯定的设计 |
| "我们提供反 GEO 监测" | ✅ 真做的话是 ⭐⭐⭐ 方向（元假说 C）；当前几乎无人在做 |
| "我们做 GEO 战略咨询" | ✅ 真做的话是 ⭐⭐⭐ 方向；难点在交付能力（需深度懂仓库 14+1 张卡级别的内容） |
| "GEO 培训" | ⚠️ 模式合理性高（元假说 E）但客户价值争议；做与不做是判断而非对错 |
| "我们推 llms.txt 中文版 / GEO 行业标准" | ✅ ⭐⭐⭐ 长期最高 ROI 位置（元假说 D） |
| "GEO 是赛道，万亿市场" | ❌ 是商业模式包装话术，市场规模与模式合理性是两件事 |

## 六、与本仓库其他卡片的关系

本卡是仓库**供给侧**的第一张系统卡，与三张已有卡形成正交研究：

- **前置依赖 [`geo-demand-side-fitness-cn.md`](./geo-demand-side-fitness-cn.md)**——需求场景决定哪些模式能找到客户。需求只有 4 类窄场景（B2B 选型 / YMYL / 长尾 / 跨语言），意味着 c/d 层的"通用 GEO"模式天然缺市场，必须垂直化（元假说 F）
- **互补 [`geo-roi-framework-cn.md`](./geo-roi-framework-cn.md)**——后者是单方决策视角（客户该不该投），本卡是供给侧视角（厂商该建什么模式）。两卡共用 C-SEO Bench 锚点
- **引用 [`platform-search-blockade-cn.md`](./platform-search-blockade-cn.md)**——平台壁垒结构是预判平台原生广告位（元假说 A）的基础
- **引用 [`citation-attribution-styles-cn.md`](./citation-attribution-styles-cn.md)**——引用源 CTR < 1% 直接限制流量分成型变现（部分 c 层模式）
- **引用 [`private-vs-public-source-cn.md`](./private-vs-public-source-cn.md)**——公众号 / 知乎 / 百科代运营矩阵的有效性边界，决定 c 层细分模式的客户价值
- **引用 [`overseas-cn-deployment.md`](./overseas-cn-deployment.md)**——跨语言陪跑（元假说 F 子场景）的具体执行框架

衍生回填到 [`dimension-map-cn.md`](./dimension-map-cn.md) 的建议：把 8 维度更新为 **D0（需求侧）+ D1-D8（信号侧）+ DS（供给侧 / 商业模式侧）** 的三轴结构。

## 七、不做的事

- 不做"哪个模式利润最高"排名——多数财务数据非公开，估值噪声大
- 不做"创业建议"——本仓库立场是研究而非创业指导
- 不预测某家具体公司命运（探词或其他）——研究商业模式而非具体公司
- 不做"是否值得投资 GEO 公司"的投资分析
- 不评判"客户价值低 + 模式合理"组合的道德性——焦虑变现是描述而非对错
- 不做"如何抢占某层" 的执行手册——这反过来变成本仓库要拆解的对象

## 八、待解决的未知 / 公开数据空白

1. **中国 GEO 公司真实收入构成**（SaaS vs RaaS vs OEM 占比）——元假说 B 的核心数据空白
2. **客户复购率 / 续约率**——直接反映客户价值，但厂商极少披露
3. **平台原生广告位的时间表**——豆包 / 元宝 / 千问 / 文心 是否在内部测试，无独立信息
4. **反 GEO 监测的真实需求规模**——元假说 C 的核心数据空白；公关 / 法务部门预算分配数据稀缺
5. **培训认证模式的市场上限**——SEO 培训行业历史规模可参考但中国 GEO 起点不同
6. **标准 / 协议层中国推动方的进展**——目前看不到任何公开动作，可能空白本身就是机会
7. **垂直陪跑公司 vs 通用 GEO 公司的存活率差异**（元假说 F）——需要 3-5 年纵向跟踪
8. **OEM / 源码模式的市场扩散速度**——58888 元源码（探词）后是否会出现更便宜的 1-3 万版本
