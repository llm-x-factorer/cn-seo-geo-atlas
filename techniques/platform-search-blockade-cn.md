# 中文平台间搜索 / 跳转 / 收录的相互封锁

> ⚠️ **状态：concept**（封锁事实多数有历史实证，但**当前**状态因互联互通政策动态变化，且各平台细则**不公开**——本卡是当前可观察事实的整理，不是历史档案）
>
> 这张卡的核心叙事是**英文圈完全空白**的中文场景特殊性——英文场景的开放 Web 是默认假设，但中文场景的"信源池"是被多年平台间相互封锁切割成的**几个互不连通的子池**。
>
> 本卡做四件事：
>
> 1. **拆开"封锁"**——这是个伞形术语，至少包含 5 类不同的技术 / 商业行为
> 2. 整理 2008-2026 关键历史节点的时间线（**事实层**），不评判对错
> 3. 给出 2026 当前的封锁状态矩阵（**当前事实**），需要持续维护
> 4. 拆解封锁结构对**中文 LLM 信源池**的影响——这是英文 awesome list 完全没讨论的核心问题
>
> ⚠️ **怀疑锚点 1**：本卡多数事实陈述基于**长期行业观察**——但具体规则细节（例如某 robots.txt 第几行、某条 API 何时开放）会随时间变化。**引用本卡时请做时点核查**。
>
> ⚠️ **怀疑锚点 2**：本卡不评判"封锁是否合理"——这是商业 + 监管 + 反垄断政策的叠加结果，本仓库只描述**结构后果**对 SEO/GEO 的影响。
>
> ⚠️ **怀疑锚点 3**：[C-SEO Bench](https://openreview.net/forum?id=oTeixD3oZO) 与本卡的关联——封锁结构造成中文 LLM 信源池的**生态化分裂**（豆包看到的 Web ≠ 文心看到的 Web ≠ DeepSeek 看到的 Web），这是英文场景没有的复杂度，使得"GEO 通用打法"的话术更值得怀疑。

## 一、"封锁"是伞形术语——5 类不同行为

服务商话术常笼统讲"X 屏蔽了 Y"，但这个表述至少包含 5 个独立维度：

| 维度 | 描述 | 技术形态 |
|------|------|----------|
| **C1 爬虫屏蔽** | A 站点用 robots.txt / 反爬虫拒绝 B 搜索引擎抓取 | robots.txt + UA 黑名单 + 频率限制 |
| **C2 索引屏蔽** | A 搜索引擎主动不收录 B 平台内容（即使技术上能爬） | 搜索引擎一侧的过滤规则 |
| **C3 跳转屏蔽** | A 平台内不允许跳转到 B 平台的链接（点击 → 提示"无法直接打开"） | URL 白名单 + 跳转拦截 |
| **C4 内容引用屏蔽** | A 平台搜索内不展示 B 平台来源 | 搜索结果过滤 / 来源白名单 |
| **C5 API 不开放** | A 平台不向第三方提供 API 抓取 / 内容获取 | API 白名单 / 商业合作门槛 |

**关键观察**：5 类封锁的**技术成本 + 商业含义 + 反垄断审查标准都不同**——讨论封锁时必须明确是哪一类。

## 二、关键历史节点时间线（2008-2026）

> 时间线为公开行业观察整理，**具体节点需对照官方公告 + 媒体报道核实**。

### 2.1 阶段一：2008-2018（默认封锁建立期）

| 年份 | 事件 | 封锁类型 |
|------|------|----------|
| 2008 | 淘宝 robots.txt 禁止百度爬虫，阿里启动自家"一淘"搜索 | C1 + C2 |
| 2009-2012 | 微信公众号生态崛起，文章默认不进入百度索引 | C2 |
| 2010-2014 | 微博 vs 微信生态对峙；微博内容部分进入百度，微信几乎不进 | C2 |
| 2013-2015 | 知乎崛起，初期内容公开收录于百度，后期部分内容只在站内 | C2（部分） |
| 2014-2016 | 字节系（今日头条）拒绝百度收录，自建头条搜索 | C1 + C2 |
| 2018 | 抖音崛起，抖音视频被腾讯系限制跳转 | C3 |

### 2.2 阶段二：2018-2021（生态对垒激烈期）

| 年份 | 事件 | 封锁类型 |
|------|------|----------|
| 2018-2020 | 字节系 × 腾讯系互诉（抖音诉腾讯封禁、腾讯诉抖音 / 头条爬取微信内容等） | C3 + C4 + C5 |
| 2019 | 抖音外链在微信内打不开（"无法直接打开"提示）成行业常态 | C3 |
| 2020 | 拼多多 / 美团等链接在抖音内的跳转限制 | C3 |
| 2020 | 字节系收购原互动百科 → 抖音百科，整合自家百科生态 | 生态闭环建立 |
| 2021-03 | 国家市场监督管理总局发布《关于平台经济领域的反垄断指南》 | 政策启动 |

### 2.3 阶段三：2021-2026（互联互通政策推进期）

| 年份 | 事件 | 封锁类型 |
|------|------|----------|
| 2021-09 | 工信部召开"屏蔽网址链接问题行政指导会"，要求平台不得封禁外链 | 政策强制 |
| 2021-11 起 | 微信开始**部分**允许外链跳转（点对点聊天场景先行） | C3 部分放开 |
| 2022-2023 | 各平台陆续修订 robots.txt / 跳转规则，整体趋势放开 | C1 + C3 部分放开 |
| 2023-2024 | 但 C2 索引屏蔽（搜索引擎不主动收录其他平台内容）**几乎未变化** | C2 仍在 |
| 2024-2026 | 生成式 AI 兴起，封锁结构在 LLM 训练数据 + RAG 召回层重新表现 | 新维度浮现 |

**关键观察**：互联互通政策**主要解决了 C3 跳转屏蔽**——但 **C1 爬虫 + C2 索引 + C5 API 这三类封锁多数仍在**。LLM 训练数据 + RAG 召回的核心受限在 C1 / C2 / C5，所以"互联互通"对 GEO 维度的实际改善有限。

## 三、2026 当前封锁状态矩阵（行业可观察事实）

> 矩阵为**当前可观察的封锁状态**——具体细节会变化，**引用前请重新核实**。

### 3.1 主要封锁关系一览

行：被封锁方，列：封锁方。`✓` 存在该类封锁，`部分`、`已放开`、`未知`。

| 被封锁 \ 封锁 | 百度 | 微信 | 字节系（今日头条 / 抖音） | 阿里（淘宝 / 1688） |
|--------------|------|------|------------------------|--------------------|
| **百度爬虫抓取淘宝** | — | — | — | **C1 ✓**（robots.txt 长期屏蔽） |
| **百度收录公众号文章** | **C2 ✓**（微信侧不开放给百度爬取） | — | — | — |
| **百度收录头条 / 抖音** | **C2 部分**（字节系自家搜索为主） | — | — | — |
| **微信内跳转抖音 / 淘宝** | — | **C3 部分放开**（外链已可跳转）| — | — |
| **百度内嵌结果显示淘宝商品** | — | — | — | **C4 ✓**（淘宝搜索结果不出现在百度商品类） |
| **微信搜一搜显示抖音视频** | — | **C4 部分**（不主动展示字节系） | — | — |
| **小红书内搜索抖音内容** | — | — | — | — / 小红书有自家搜索逻辑，跨平台几乎不展示 |
| **抖音内搜索 B 站 / 知乎内容** | — | — | **C4 ✓** | — |
| **视频号内显示 B 站 / 抖音内容** | — | **C4 ✓** | — | — |

### 3.2 几条值得单列的现状

- **公众号文章在百度搜索**：长期不可见。微信只允许微信搜一搜搜公众号；百度爬虫被微信侧拦截。这是 C1 + C2 的复合封锁
- **淘宝商品在百度**：长期 C1（robots.txt 屏蔽）+ C4（百度展示阿里电商有协议限制）双重封锁
- **抖音视频在微信搜一搜**：C4 屏蔽——微信搜一搜默认不展示字节系视频。视频号内嵌为腾讯系替代
- **百家号 vs 头条号**：表面互不收录（C2），实际两家自有 LLM（文心 vs 豆包）通过自家生态绕过对方
- **小红书 vs 抖音**：互为内容竞品，搜索结果几乎完全分裂，但站外用户搜索（在百度搜"小红书 X"或"抖音 X"）仍部分可达

### 3.3 互联互通已放开的部分

- **聊天场景的外链跳转**（微信内 1V1 / 小群可点开外链）—— C3 部分解除
- **政府类 / 公益类外链**——行政强制开放
- **部分电商促销外链**（如双 11 期间互通）—— 临时性

**关键观察**：**用户体验层**的互联互通有所改善（C3），但**搜索引擎 + LLM 信源池层**的封锁结构（C1 + C2 + C5）几乎没变——这是 GEO 维度上必须分清的两件事。

## 四、封锁结构对中文 LLM 信源池的影响

每个中文 LLM 看到的"Web"实际上是被封锁结构切割的**子池**：

| LLM | 主信源池 | 受封锁影响 | 备注 |
|-----|----------|------------|------|
| 文心 | 百度索引 + 百度生态（百家号 / 百科 / 文库 / 知道） | 看不到淘宝 / 公众号 / 抖音 / 头条主体内容 | 受 C1 + C2 影响最大 |
| 豆包 | 字节系生态 + 通用爬取 | 看不到公众号主体 / 淘宝主体 | 字节生态闭环但腾讯 / 阿里方面被锁 |
| 元宝 | 腾讯生态（公众号 / 视频号 / 微信搜一搜）+ 通用爬取 | 看不到字节系核心 / 淘宝主体 | 腾讯生态闭环但字节 / 阿里方面被锁 |
| 千问 | 阿里电商生态 + 通用爬取 | 公众号 / 字节系受限 | 商品类查询占优势 |
| DeepSeek | 通用爬取，无生态绑定 | **被所有平台封锁影响**——爬到的是 L1 公开 Web 的边角 | 公开度 L1 偏向 |
| Kimi | URL 抓取 + 通用爬取 | 同 DeepSeek，再加用户上传 | 用户输入触发为主 |

**核心后果**：

- 文心 / 豆包 / 元宝 / 千问 = **生态闭环 LLM**，每家看到不同的 Web
- DeepSeek / Kimi = **通用 LLM**，看到的是被各家封锁后的 L1 残余 Web
- **没有任何中文 LLM 看到完整的中文 Web**——这与英文场景下 GPT / Claude / Gemini 大体看到相同公开 Web 形成结构差异

## 五、待验证元假说

### 元假说 A：中文 LLM 之间的信源池 Jaccard 相似度 < 50%

- 表述：5 大中文 LLM 在同一 query 下召回信源 URL 的 Jaccard 相似度均值 < 0.5——即重合不足一半
- **驱动力**：封锁结构造成各 LLM 看到不同的 Web 子池
- **如何验证**：[`exp-five-vendor-paired-queries`](../experiments/exp-five-vendor-paired-queries.md) 数据可直接析出 Jaccard 维度
- **关联**：[`prompt-level-seo-cn.md`](./prompt-level-seo-cn.md) 元假说 A（信源池差异 > 30%）的进一步具体化

### 元假说 B：通用 LLM（DeepSeek / Kimi）的中文信源池**显著偏向 L1 公开站**

- 表述：DeepSeek / Kimi 召回信源中 L1 占比显著高于豆包 / 元宝 / 千问 / 文心；C1 / C2 受封锁内容（公众号 / 淘宝 / 抖音）占比接近 0
- **驱动力**：通用爬取无法绕过封锁
- **如何验证**：从五大厂实验数据按公开度档位 + 平台来源分类
- **关联**：[`private-vs-public-source-cn.md`](./private-vs-public-source-cn.md) 元假说 A 的姊妹命题

### 元假说 C：互联互通政策（2021-2026）对 LLM 引用率的实际影响**接近 0**

- 表述：2021 年互联互通政策主要解决 C3 跳转屏蔽——这一层对 LLM 训练 / RAG 几乎无关；C1 + C2 + C5 三类核心封锁未实质改变
- **含义**：内容方期待"互联互通会让我的内容被所有 AI 引用"是**误读政策**——政策意图在用户体验，不在 LLM 信源池
- **如何验证**：对比 2021 年前后中文 LLM 召回信源池结构的变化（数据缺失）

### 元假说 D：跨生态发布是中文 GEO 唯一覆盖多 LLM 的可行路径

- 表述：内容方在自家生态对应的"号"上发布（百家号 + 头条号 + 公众号 + 知乎 + 小红书 + B 站等）是**绕过封锁**触达多 LLM 的**唯一**操作路径
- **驱动力**：封锁是结构性的，单一信源载体无法覆盖
- **关联**：[`account-grade-systems-cn.md`](./account-grade-systems-cn.md) 元假说 E（跨生态号矩阵）的封锁视角解释——后者讲"为何要做矩阵"，本卡讲"为何不能不做矩阵"
- **副作用**：本假说成立则**中文 GEO 的工程量结构性高于英文 GEO**——这是中小品牌的硬门槛

### 元假说 E：服务商"互联互通后做一处覆盖所有 AI"话术是**对政策误读**

- 表述：互联互通政策范围限于 C3，**没有触及 C1 / C2 / C5**；服务商以"互联互通"为依据宣称"做一处全覆盖"是误读 + 话术
- **如何验证**：直接读 2021-09 工信部行政指导会公开的范围 + 后续监管文件——即"屏蔽网址链接"是 C3 范畴

## 六、实验设计建议

### 实验 1：跨 LLM 信源池 Jaccard 量化（核心实验）

- **目标**：验证元假说 A
- **设计**：从 [`exp-five-vendor-paired-queries`](../experiments/exp-five-vendor-paired-queries.md) 数据析出
- **优先级**：⭐⭐⭐ 高（无需新数据采集）
- **关联**：与 [`prompt-level-seo-cn.md`](./prompt-level-seo-cn.md) 的实验 1 是同一份数据的不同切片

### 实验 2：通用 LLM vs 生态 LLM 的信源平台分布对照（验证元假说 B）

- **目标**：量化"通用 LLM 看到的中文 Web"与"生态 LLM 看到的中文 Web"差异
- **设计**：按平台域名分类（公众号 / 淘宝 / 抖音 / 头条 / 公开独立站等）统计 5 大 LLM 的引用占比
- **复用**：扩展 [`tools/exp-doubao-vs-deepseek-paired/source_classifier.py`](../tools/exp-doubao-vs-deepseek-paired/source_classifier.py) 加入封锁来源标签
- **优先级**：⭐⭐⭐ 高

### 实验 3：跨生态发布 vs 单生态深耕的 5 平台覆盖率（验证元假说 D）

- **目标**：直接量化"跨生态矩阵"对覆盖率的边际收益
- **设计**：与 [`account-grade-systems-cn.md`](./account-grade-systems-cn.md) 实验 2 同源——可联合设计
- **优先级**：⭐⭐ 中

### 不建议做的实验

- **"互联互通后某平台外链跳转 CTR 变化"**：与 LLM 引用率因果链最远，仓库不收录
- **"测试某平台是否封锁某 UA"**：法律边界灰色（涉及反爬虫机制对抗），仓库不做

## 七、与 GEO 服务商话术的对照

国内常见话术：

- "互联互通后做一处覆盖所有 AI"
- "我们打通了所有平台的内容互通"
- "封锁已成历史，单一独立站即可"
- "用我们的工具绕过平台封锁"

**本卡片判定**：

| 话术 | 判定 | 理由 |
|------|------|------|
| "互联互通后一处覆盖所有 AI" | ❌ 政策误读 | 元假说 C + E；C1/C2/C5 未触及 |
| "打通所有平台" | ❌ 商业不可能 | 各平台资本独立，不会单方"打通" |
| "封锁已成历史" | ❌ 仅 C3 部分放开 | 时间线显示 C1+C2+C5 仍在 |
| "工具绕过封锁" | ❌ 法律灰色 + 不可持续 | 反爬虫对抗本身违反平台条款 |
| "做生态矩阵覆盖多 LLM" | ✅ 与元假说 D 一致 | 这是**结构上正确的**唯一路径 |
| "无品牌靠通用 LLM 流量" | ⚠️ 元假说 B 揭示其偏向 L1 残余 Web | 通用 LLM 流量在中文场景天然受封锁结构限制 |

**结论**：服务商话术里**与"跨生态矩阵"对齐**的部分基本上是结构正确的；**承诺"打通封锁"或"互联互通后单点覆盖"**的部分都是误读 + 不可能的承诺。

## 八、与本仓库其他卡片的关系

- 与 [`private-vs-public-source-cn.md`](./private-vs-public-source-cn.md)：本卡是该卡的**因果上游**——L2 半公域信源对外部 LLM 的不可达性，根源在 C1 + C2 + C5 类封锁
- 与 [`account-grade-systems-cn.md`](./account-grade-systems-cn.md) 元假说 E：跨生态号矩阵是中文 GEO 必要工程——本卡补充"为何必要"的封锁结构解释

## 九、不做的事

- **不评判平台封锁是否合理**——这是商业 + 反垄断政策叠加结果，本仓库只描述结构后果
- **不提供"绕过封锁"的技术方案**——反爬虫对抗违反平台条款 + 法律灰色
- **不复述各平台 robots.txt 完整规则**——直接读 robots.txt 更准（且变化频繁）
- **不预先断言互联互通政策的未来走向**——政策动态，避免推测
- **不进入"哪个平台是封锁始作俑者"的归因争论**——多方互相为因，归因不明确
