# 中文场景的"号"主体绑定与账号等级体系（跨平台横向对照）

> ⚠️ **状态：concept**（横向对照卡，平台等级体系的存在事实清晰，**但等级 → 可见性 / 引用率的因果效应量级未实证**）
>
> 这是 [`techniques/eeat-cn.md`](./eeat-cn.md) 的元假说 E（"号"主体绑定权重 ≥ E-E-A-T 任一维度）的展开。E-E-A-T 卡片只点到，本卡做横向对照。
>
> 本卡做三件事：
>
> 1. 把 "号" 体系作为**中文场景独有的硬信号**单列出来（英文 E-E-A-T 框架不覆盖）
> 2. 横向对照 6+ 平台的认证 / 等级机制——确认事实，**不评判优劣**
> 3. 把"账号等级 → SEO 可见性 → LLM 引用率"的因果链拆成 3 段，分别标注哪段已实证、哪段是假说
>
> ⚠️ **怀疑锚点 1**：账号等级 / 认证体系的存在是**事实**，但"等级提升 N 档 → 可见性提升 X%"是**服务商话术**——目前没有任何中文平台公开过等级权重的具体数值，跨平台对照只能做"有 / 无 / 强 / 弱"四档定性。
>
> ⚠️ **怀疑锚点 2**："账号等级影响平台**内部**分发"和"账号等级影响**外部 LLM** 引用率"是两回事。前者各平台官方文档普遍承认（虽不给数值），后者**完全是假说**——LLM 训练数据爬取 / RAG 检索召回**未必能感知账号等级**，多数情况下抓的是裸 HTML 内容。
>
> ⚠️ **怀疑锚点 3**：[C-SEO Bench](https://openreview.net/forum?id=oTeixD3oZO) 在英文场景发现"权威性"类信号在严格控制下边际效应远低于宣称——"高等级号代发"这一中文 GEO 服务热门话术的实际效应量级在中文 LLM 上未有任何严格实证。

## 一、什么是"号"主体绑定 + 账号等级体系

### "号"主体绑定

中文内容平台的**基本单位是"号"**，不是个人或站点：

- 百度生态：**百家号**
- 字节生态：**头条号** / **抖音号**
- 腾讯生态：**微信公众号** / **微信视频号**
- 阿里生态：**淘宝商家账号** / **1688 账号**
- 第三方独立：**知乎账号** / **小红书账号** / **B 站账号** / **微博账号**

每个"号"绑定一个**主体**（个人 / 企业 / 政府机构 / 媒体），主体审核是开号前置门槛。一个主体可在不同生态各开一个号，但**号之间不互通**——百家号 V 等级不会传导到头条号。

### 账号等级体系

每个"号"在自家生态内有**多套并行的等级 / 认证标签**，至少包括：

- **主体认证**（蓝 V / 黄 V / 政府 V / 媒体 V）——一次性资质门槛
- **原创认证**——内容生产能力门槛
- **活跃 / 互动等级**——动态指标，反映持续生产
- **专业身份认证**（医师 / 律师 / 教师等）——YMYL 类专项
- **创作激励 / 商业合作分级**（蒲公英、星图、巨量星图等）

这些标签**叠加**作用，没有"统一总分"。

## 二、为什么这是中文场景独有

| 维度 | 英文场景 | 中文场景 |
|------|---------|----------|
| 内容生产基本单位 | **站点**（域名 + author byline） | **号**（账号主体 + 平台绑定） |
| 站点 vs 账号权重 | 站点权重 >> 个人作者 | 号权重 >> 站点（站点常常根本不存在） |
| 认证机制 | 平台内（Twitter Verified / LinkedIn Premium）属社交身份，不进入搜索 / LLM 信号 | 平台主体认证（蓝 V / 黄 V）**直接影响**内部分发权重，并经由训练 / RAG 间接进入 LLM |
| 等级数值 | 一般不公开 | 一般不公开**或**部分公开（如知乎盐值、B 站 LV0-LV6）但**算法权重不公开** |
| 可移植性 | 站点权重可跨搜索引擎传导 | 号权重**强绑定生态**——百家号高 V 在字节系无意义 |
| 监管耦合 | 弱（平台自治） | **强**（ICP 备案 + 主体认证 + 专项资质受行政监管） |

**核心差异**：英文 SEO 优化"站点"（domain authority），中文 SEO/GEO 优化"号"（多生态铺设）。这是 [`eeat-cn.md`](./eeat-cn.md) 元假说 D 的具体表现——服务商话术里的"提升 E-E-A-T"在中文场景实际是"做号"。

## 三、6+ 平台账号等级体系横向对照

> 表中**等级数值若公开则给出**，**算法权重一律未公开**——这是事实陈述，不是评判。

### 3.1 各生态主流"号"概览

| 生态 | "号"名称 | 主体认证（已知形态） | 原创认证（已知形态） | 等级体系（已知形态） | 专业身份认证 |
|------|----------|--------------------|---------------------|---------------------|--------------|
| 百度 | **百家号** | 蓝 V（企业 / 媒体 / 政府）+ 黄 V（个人） | 原创标识 | 历史曾有 V1-V8 数字等级（具体规则随版本调整） | 医疗 / 法律 / 教育等专项 |
| 字节—资讯 | **头条号** | 蓝 V / 黄 V | 原创权益 | 兴趣认证 + 健康号 / 财经号等垂直认证 | 医疗 / 法律 / 财经等 |
| 字节—短视频 | **抖音号** | 企业号（蓝 V）/ 个人认证 | 原创视频标识 | 达人等级（粉丝量 / 互动量挂钩，巨量星图分级）| 医疗 / 政务 / 教育 |
| 腾讯—图文 | **微信公众号** | 蓝 V（企业 / 政府 / 媒体认证）+ 个人主体 | 原创声明 + 白名单转载 | 阅读量 / 历史合规为隐性指标，平台不公开数字等级 | 政务号 / 医疗号 / 教育号专项 |
| 腾讯—视频 | **微信视频号** | 蓝 V + 个人实名 | 原创声明 | 创作者等级 + 流量主分级 | 医疗 / 政务 / 教育 |
| 阿里—资讯 | **大鱼号 / UC 头条号** | 蓝 V / 黄 V | 原创权益 | 兴趣认证 + 平台合作分级 | 健康 / 法律等垂直 |
| 知乎 | **知乎账号** | 机构号 + 优秀回答者 + 专业身份认证 | 原创声明 | **盐值**（公开数值，5 维度合成）+ 优秀回答者徽章 | 医师 / 律师 / 学者等（线下资质审核） |
| 小红书 | **小红书账号** | 专业号 / 品牌号 / 个人 | 原创声明 | 用户成长等级 + 蒲公英达人等级（合作分级） | 医师 / 营养师等专项 |
| B 站 | **B 站 UP 主** | 个人实名 / 企业认证 | 原创 / 转载二选一 | **LV0-LV6 用户等级**（公开，按经验值合成）+ 创作激励 + 充电分级 | 知识区认证 |
| 微博 | **微博账号** | 蓝 V / 黄 V / 红 V | 原创 / 转发标识 | 微博等级 + 头部账号 / V+ 会员 | 媒体 / 政务 / 行业 |

### 3.2 共同模式（去掉名词差异后的结构）

每家平台的等级体系都至少包含 3 层：

| 层级 | 性质 | 中文场景示例 | 是否影响 SEO/GEO |
|------|------|--------------|-----------------|
| **L1 主体合规** | 一次性资质门槛 | ICP 备案、主体认证、行业资质（医师 / 律师 / 持牌金融） | **强**——多数生态对未通过 L1 的内容直接降权或不收录 |
| **L2 原创身份** | 内容生产能力声明 | 原创标识 / 原创号 / 转载白名单 | **中**——影响生态内分发，对外部 LLM 影响未知 |
| **L3 活跃 / 互动等级** | 持续表现指标 | 百家号历史 V 等级、知乎盐值、B 站 LV、抖音达人分级 | **弱到中**——主要影响生态内推荐，对 SEO/GEO 影响最不确定 |

**关键观察**：服务商话术里"提升账号等级"通常指 L3，但 SEO/GEO 的实际可见性差异**主要来自 L1**——L1 是离散开关（有 / 无），L3 是连续刻度，前者效应量级远大于后者。

## 四、账号等级 → 中文搜索引擎可见性（已知 / 推测）

### 4.1 百度搜索

- 百家号在百度搜索中享有结构性优先权（公开事实，详见 [`platforms/baidu/seo-signals-and-ai-overview.md`](../platforms/baidu/seo-signals-and-ai-overview.md)）
- 百家号 V 等级与收录 / 排名权重的关系：**官方未公开数值**；服务商口径"V4 以上享有快速收录"是**经验声称**，无严格实证

### 4.2 神马 / 360 / 搜狗

- 神马属阿里系，对**大鱼号 / UC 头条号**有生态偏好（推测）
- 360 / 搜狗的账号体系与搜索权重耦合度更弱（推测）

### 4.3 微信搜一搜

- 微信公众号 + 视频号在微信搜一搜里**结构性权重显著**——这是搜一搜唯一的差异化信源池
- 蓝 V 认证 vs 个人主体在搜一搜结果排序里的差异：未公开数值

### 4.4 内容平台站内搜索

- 小红书：见 [`platforms/xiaohongshu/on-platform-search.md`](../platforms/xiaohongshu/on-platform-search.md)，账号等级 + 历史互动是已知信号
- 知乎站内搜索：盐值与排名相关性公开承认（"对盐值高的用户内容做小幅加权"），具体权重未公开
- 抖音 / B 站：站内搜索的账号信号权重未公开

## 五、账号等级 → 中文 LLM 引用（推测）

> ⚠️ **本节全部为推测**——5 大中文 LLM 厂商**均未公开**训练数据筛选 / RAG 召回时是否使用账号等级信号。

### 5.1 三段因果链拆解

`账号等级 ↑` → `平台内部分发权重 ↑` → `内容曝光 / 引用 / 转载量 ↑` → `被 LLM 抓取 / 检索召回概率 ↑` → `生成答案中被引用 ↑`

| 链段 | 实证状态 |
|------|----------|
| 等级 → 平台内部分发 | 平台官方普遍承认（无具体数值） |
| 内部分发 → 内容曝光 | 强相关（基本事实） |
| 内容曝光 → 被 LLM 召回 | **未实证**（与训练数据 / RAG 实现耦合） |
| 被召回 → 被引用 | **未实证**（与生成策略耦合） |

**只有把这四段全部拆开，才能避免把"等级 → 引用率"当成单一因果**。多数 GEO 服务商话术省略中间两段。

### 5.2 各 LLM 推测

| LLM | 是否可能感知账号等级 | 理由 |
|-----|-------------------|------|
| 文心 | **较高**——共用百度搜索索引时大概率继承百家号权重 | 与底层搜索 E-E-A-T 类信号最耦合的 LLM |
| 豆包 | 中——字节系生态内（头条号 / 抖音号）有路径感知，外部内容裸抓 | 详见 [`platforms/doubao/citation-mechanism.md`](../platforms/doubao/citation-mechanism.md) |
| 元宝 | 中——微信公众号原创标识可能在生态内传导 | 微信生态外通用爬取 |
| 千问 | 中—弱——电商类对商家等级有信号；非电商类未知 | 详见 [`platforms/qianwen/citation-mechanism.md`](../platforms/qianwen/citation-mechanism.md) |
| DeepSeek | 弱——通用爬取，无生态绑定 | 等级信号大概率丢失 |
| Kimi | 极弱——主要 URL 抓取触发，账号信号最难传导 | 长文本场景，对账号属性几乎无依赖 |

**核心推测**：账号等级对 LLM 引用率的影响**仅在生态内传导**——文心 / 百家号、豆包 / 头条号、元宝 / 公众号是有可能感知到的；其他配对几乎不可能。

## 六、待验证元假说

### 元假说 A：L1 主体合规的可见性效应量级 >> L3 活跃等级

- 表述：在所有平台上，"无主体认证 → 有主体认证"的可见性提升量级**显著大于** "L3 等级 N → N+1 档"的可见性提升
- **含义**：投入资源做主体合规（一次性）的 ROI 远高于刷等级（持续投入）
- **如何验证**：受控发布同质内容到 4 类账号——`未认证个人`、`已认证个人`、`蓝 V 低等级`、`蓝 V 高等级`，比较 SEO 收录率 + LLM 引用率
- **关联**：[`eeat-cn.md`](./eeat-cn.md) 元假说 E

### 元假说 B：账号等级对外部 LLM 引用率的影响**仅在自家生态内传导**

- 表述：百家号高 V → 文心引用率 ↑；头条号高 V → 豆包引用率 ↑；公众号原创 → 元宝引用率 ↑——**跨生态时这种关系消失**
- **含义**：服务商话术"高等级号代发可提升所有 LLM 引用率"假设了"等级 → 引用率"是平台无关的，本假说反对
- **如何验证**：同主题内容分别从（百家号 / 头条号 / 公众号）3 个高等级号发出，跑 5 大 LLM 看引用率是否在对应生态显著提升而在其他生态不显著
- **关联**：[`eeat-cn.md`](./eeat-cn.md) 元假说 B、[`princeton-9-strategies-cn.md`](./princeton-9-strategies-cn.md) 元假说 B

### 元假说 C："专业身份认证"在 YMYL 类的效应远高于普通认证

- 表述：YMYL 类 query（医疗 / 法律 / 金融）下，"持医师 / 律师 / 持牌金融认证的号"的引用率显著高于"普通蓝 V"——非 YMYL 类则无显著差异
- **含义**：YMYL 类不靠"做号"靠"做资质"——这是 [`eeat-cn.md`](./eeat-cn.md) 元假说 C 的"号"侧表现
- **如何验证**：YMYL 子集 query 跑跨平台对照，析出"专业认证号 / 普通号 / 无认证"三档引用率

### 元假说 D：L3 活跃等级（数字等级）对外部 LLM 引用**几乎无效应**

- 表述：百家号 V4 vs V8 / 知乎盐值 600 vs 800 / B 站 LV4 vs LV6 在外部 LLM 引用率上**差异 < 5pp**（弱效应阈值）
- **含义**：刷等级是平台内分发优化，**不是 GEO 优化**——LLM 抓取的是裸 HTML，等级信息多半丢失
- **副作用**：服务商"提升账号等级套餐"在 GEO 维度多为无效投入（平台内分发维度可能仍有效应）
- **如何验证**：内容质量对齐前提下，比较同一作者账号在等级跃升前后 90 天的 LLM 引用率变化

### 元假说 E：跨生态"号矩阵"是中文 GEO 的真正硬信号

- 表述：单一高等级号 < 多生态中等级号矩阵（百家号 + 头条号 + 公众号 + 知乎 + 小红书同时具备）
- **含义**：把元假说 B 的反向推论——既然等级仅在生态内传导，那么覆盖更多生态的边际收益高于深耕单生态
- **结构性壁垒**：跨生态铺设需要不同主体 / 资质 / 时间投入，是中小品牌的实际门槛
- **如何验证**：长周期对照——同主题在 1 个高等级号集中发布 vs 5 个中等级号矩阵发布的 90 天 LLM 引用率累积

## 七、实验设计建议

### 实验 1：主体认证 vs 等级数值的效应分离（核心实验）

- **目标**：验证元假说 A + D（L1 强 / L3 弱）
- **设计**：
  - 自变量：4 档（未认证个人 / 已认证个人 / 蓝 V 低等级 / 蓝 V 高等级）
  - 因变量：搜索收录率 + 5 大 LLM 引用率
  - 控制：内容主题、长度、发布时间、平台（建议先单平台百家号或头条号）
- **样本**：每档 ≥ 5 篇内容 × 30 天观察 = 单平台 4×5×30 共 600 数据点
- **难点**：需 4 类真实账号（合规获取，有资源门槛）
- **优先级**：⭐⭐⭐ 最高
- **关联 BACKLOG**：`exp-cn-account-cert-impact`（已在 [`eeat-cn.md`](./eeat-cn.md) 衍生）

### 实验 2：跨生态号矩阵 vs 单号深耕（验证元假说 E）

- **目标**：跨生态铺设的边际收益是否真的高于单生态深耕
- **设计**：A 组同主题内容仅在 1 个高等级号发布（百家号 V6+）；B 组同主题内容在 5 个中等级号发布（百家号 V3 + 头条号原创 + 公众号原创 + 知乎 + 小红书各一）
- **因变量**：90 天累积的 5 大 LLM 引用率 + 搜索可见性
- **优先级**：⭐⭐ 中（资源消耗大）

### 实验 3：等级跃升前后引用率对照（验证元假说 D）

- **目标**：等级数字提升是否产生 LLM 引用率的可观测变化
- **设计**：长期跟踪自然存在的等级跃升事件（如百家号从 V4 升 V5），对比跃升前 60 天 vs 跃升后 60 天的引用率
- **优势**：不需要新创建账号，可被动观察
- **风险**：观察事件时多变量混淆（内容质量、平台政策同期变动）

### 不建议做的实验

- **"提升账号等级 N 档可提升 LLM 引用 X%"**：这种命题假设了线性单变量关系，与实证结构不符——应拆成 L1 / L2 / L3 三层分别测
- **"高等级号代发对照实验"**：服务商常见话术，但代发场景下"内容质量 + 主体身份"高度耦合，难以单独归因到等级
- **跨生态 6 平台全对照单实验**：变量过多导致样本不足，建议先做实验 1 的单平台版本，再扩展

## 八、与 GEO 服务商话术的对照

国内常见话术：

- "提升百家号 V 等级，保证 AI 引用率"
- "高权重账号代发，文心 / 豆包必收"
- "我们持有 N 个 V8 大号，可以快速被 AI 引用"
- "买媒体号矩阵，覆盖所有 AI 平台"

**本卡片判定**：

| 话术 | 判定 | 理由 |
|------|------|------|
| "等级 → 引用率"线性可预测 | ❌ 因果链拆解后至少 4 段，每段都有大量未实证 | 第五节因果链拆解 |
| "高等级号代发必收" | ⚠️ 在自家生态内可能成立，跨生态大概率失效 | 元假说 B |
| "买媒体号矩阵覆盖所有 AI 平台" | ⚠️ 元假说 E 部分一致——但"买"和"自己运营矩阵"不是同一回事 | 媒体号代发的内容质量与原创度信号常被平台风控削弱 |
| "提升账号等级提升 SEO 收录" | ✅ 平台内分发部分可能成立（L1 + L2 强、L3 弱） | 实验 3 待验证 |
| "提升账号等级提升 LLM 引用率" | ❌ 元假说 D 反对，未实证 | 等级是隐性元数据，LLM 抓取的是裸 HTML |

**结论**：服务商话术的部分有效部分主要在 L1 + L2（主体认证 + 原创认证）；L3 数字等级的"提升换可见性"在 GEO 维度大概率无效。

## 十、不做的事
