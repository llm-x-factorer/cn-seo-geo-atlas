# 伦理边界声明（exp-cn-rag-poisoning-datavoid-sandbox）

本工具是**防御导向的安全研究脚手架**，测量"对 RAG 投毒的检测信号有没有用"。它复刻了央视 3·15（2026）等公开案例的**威胁模型**，但严格框定在自掌控沙盒内。

## 硬约束（违反任意一条即偏离本工具用途）

1. **只在本地沙盒运行**。所有"投毒文档"只写入本地 `data/poison.jsonl` 与本地工作语料，**绝不**向任何线上第三方商用系统注入——豆包 / DeepSeek / 文心 / 千问 / 元宝 / Kimi / 百度 AI 搜索 / 任何真实网站 / 任何真实平台。

2. **不发布、不外泄**。投毒文档不入 git（在 `.gitignore`），不发到任何会被真实用户或真实爬虫检索到的位置。

3. **不损害真实主体**。`queries.yaml` 的 mature 集只用"明显荒谬的事实错误"（如珠峰高度写错），不生成可信的医疗 / 金融 / 法律类误导信息；fictional 集只用明确虚构、现实零对应的实体。

4. **故意不实现 SOTA 攻击**。`poison.py` 只生成 `natural`（流畅自然）和 `noise`（高困惑度噪声）两个**朴素**风格，**不实现** gradient-based 对抗优化（GASLITE / HotFlip / CPA-RAG 那类）。本工具要回答的是"自然文本 vs 噪声文本哪个更易被困惑度过滤抓住"，不需要、也不提供 SOTA 规避能力。

5. **rag_runner 只读不写**。`openai` / `hf` backend 只调用 LLM 生成答案，不向任何外部系统写入任何内容。

## 与仓库红线的对齐

对齐 [`CONTRIBUTING.md`](../../CONTRIBUTING.md)："研究为什么有效 / 如何被识别可以，操作手册不行。" 本工具重心在**如何被识别 / 如何防御**：它的产出是 H1（data void 决定论）/ H2（防御衰减曲线）/ H3（困惑度过滤对自然 vs 噪声文本的检出率）——全部服务于 [`anti-geo-monitoring-cn.md`](../../techniques/anti-geo-monitoring-cn.md) 的检测信号设计。

## 负责任披露范式

参照 Carlini et al.（IEEE S&P 2024）：验证攻击路径的**可行性与可检测性**，不实际投毒、不影响任何生产系统。若本沙盒发现某检测信号对中文场景特别有效或特别无效，价值在于改进防御，而非改进攻击。
