外人看 GEO 行业像是一种公司：帮品牌在 AI 答案里"被看见"。但其实 GEO 不是一种公司，至少有七种不同的生意模式，做的事、收的钱、面对的客户都不一样。

这篇博客把这七种模式拆开，每种说清楚三件事：在做什么、客户是谁、能不能长期赚钱。

理解这七层之后，你会发现：当前国内 GEO 公司多数都挤在同样的两层（c 层 + d 层），但**这两层恰恰是利润率最低、平台风险最高的位置**。

## 先说一句重要的区分

商业模式合理性 ≠ 客户价值。

这是两件独立的事：

- 客户价值：客户买了之后是否真获得效果
- 模式合理性：供给方能不能长期赚钱、能不能扩张、能不能抵御平台风险

可以这样组合：

- 客户价值高 + 模式合理：长期赢家（最罕见）
- 客户价值低 + 模式合理：合法暴利——焦虑变现 / 培训认证 / 监测仪表盘
- 客户价值高 + 模式不合理：难做大——咨询 / 高度定制服务
- 客户价值低 + 模式不合理：双重无效——多数 GEO 短命公司

下面拆七层时，会同时给出"客户价值"和"模式合理性"两个独立判断。

## 七层模式

### a 层：数据 / 监测

**做什么**：第三方监测 LLM 上的品牌可见度，做成订阅产品

类比：Similarweb 之于 SEO 行业 / 艾瑞之于互联网行业。专做"客观第三方数据"，不做优化交付。

**客户**：企业内部 BI / 营销 / 公关部门（要决策依据），媒体 / 行业研究机构（要发布报告）

**收钱方式**：订阅费（月 / 年）、API 调用费、行业报告付费

**客户价值**：高——监测是真需求且现状稀缺

**模式合理性**：⭐⭐⭐ 高——不做优化交付 = 不背效果责任 + 数据资产可累积 + 网络效应（订阅客户越多数据越精准）

**当前状态**：中文场景几乎空白。探词科技的 APIaaS 部分覆盖，但他们主业不在这

### b 层：工具 / SaaS

**做什么**：垂直工具，不做全栈

举例：专做"竞品 LLM 引用归因诊断"的 SaaS / 专做"FAQ 数据集到多平台 prompt 匹配测试"的工具 / "LLM-friendliness 检测器"等

**客户**：内容团队 / 独立站站长 / 营销技术负责人

**收钱方式**：固定 SaaS 订阅

**客户价值**：中-高——工具化解决具体问题

**模式合理性**：⭐⭐⭐ 高——开发成本可控、客户付费意愿稳定、毛利率高

**当前状态**：空白。多数厂商喜欢做"全栈"卖高单价，垂直工具反而稀缺

### c 层：内容 / 信源

**做什么**：生产内容并铺到 LLM 可能会用到的信源（百度收录站 / 公众号 / 知乎 / 小红书 / B 站 / 自建网站）

举例：探词手册里那个"集成 7 万+ 新闻媒体平台发布渠道"——这就是 c 层。本质是采购传统软文 / 新闻稿分发产业链。

**客户**：B2B / 大型品牌 / 跨境出海企业

**收钱方式**：按发布数量、按月、按效果

**客户价值**：不稳定——内容铺出去能不能真被 LLM 引用，取决于很多平台变量

**模式合理性**：⭐⭐ 中——议价权在上游媒体，毛利率被压缩

**当前状态**：中文 GEO 行业最拥挤的层。多数 GEO 服务商靠这层吃饭。但风险是"7 万+ 媒体" 大多是采购的，不是自有，平台政策一变全部受影响

### d 层：服务 / 咨询

这一层细分两种很不同的子模式：

**d1 效果型代运营（RaaS）**：

- 做什么：诊断 + 内容生产 + 信源铺设 + 监测，一条龙服务
- 客户：希望"全包"的中大型企业
- 收钱方式：基础服务费 + 效果费（按 query 位次提升计费）
- 模式合理性：⭐⭐——重人工，每个客户需要 0.5-1 个交付人，扩张曲线是线性的

**d2 战略咨询**（诊断 + 路线图，不做执行）：

- 做什么：帮客户判断要不要做 GEO、在哪些场景做、怎么算 ROI
- 客户：CMO / 战略部 / 数字化转型负责人
- 收钱方式：按项目（30 万-200 万一个项目）
- 模式合理性：**⭐⭐⭐ 高**——避开效果承诺、按项目计费、毛利高

**当前状态**：

- d1 是国内 GEO 行业当前的主要利润来源（探词的主要收入是 RaaS）
- d2 几乎空白——没有公司专做"GEO 战略咨询"

### e 层：平台 / 生态

**三种子模式**：

**e1 GEO 双边市场（客户 ⇄ 服务商对接平台）**：

- 类比：猪八戒之于设计行业
- 当前状态：空白，需要规模才能起量

**e2 行业认证 / 培训**：

- 类比：百度推广认证培训 / 巨量学
- 客户价值：低（培训对真实业务的帮助有限）
- 模式合理性：**⭐⭐ 中-高**——这是合法暴利
- 当前状态：暂时空白，但这个市场会被填满，问题只是被谁填

**e3 OEM / 白标系统**：

- 给同行卖工具
- 探词的 58888 元源码 + 贴牌服务就是 e3
- 模式合理性：**⭐ 短期可行 / 长期自伤**——同行拿到源码会做出更便宜版本（千博系统剧本，SEO 行业已经走过一遍）

### f 层：反向 / 防御

**做什么**：帮品牌识别"竞品在用 GEO 攻击我"、"AI 答案里出现虚假信息"、"舆情通过 LLM 二次传播"

**关键差异**：所有 a-e 层都是"帮品牌做 GEO"（进攻方），f 层是反过来——"帮品牌识别 / 防御 GEO 影响"

**客户**：品牌公关部 + 法务部 + 舆情监测部（**不是营销部**）

**收钱方式**：监测订阅 + 报告 + 危机响应

**客户价值**：高——公关 / 法务的预算来源不同，付费意愿稳定

**模式合理性**：**⭐⭐⭐ 高**

**当前状态**：完全空白。这是仓库识别出的最有结构性价值的方向之一。详见系列第 5 篇。

### g 层：标准 / 协议

**做什么**：推动 llms.txt / AI 内容溯源 / GEO 行业规范等协议，第一个推动的人有规则定价权

类比：Google 主导 robots.txt / sitemap.xml 等协议，长期受益

**客户**：间接——通过影响力转化为业务

**收钱方式**：影响力 + 衍生业务 + 行业组织会费

**客户价值**：高（行业基础设施）

**模式合理性**：**⭐⭐⭐ 极高**——投入低（一两个工程师 + 行业关系），产出可能极高（一旦标准被采纳，所有 GEO 公司都绕不过你）

**当前状态**：中文场景空白。《生成式人工智能服务管理办法》（2023-07）已落地，协议 / 标准窗口期开启，但暂时没有人在动手

## 七层放在一起看，得出几个判断

**判断 1：当前国内 GEO 行业拥挤的两层是 c + d，但这两层利润率最低**

c 层议价权在上游媒体，d1 是重人工咨询模式。多数玩家挤在这两层是因为"看起来好做、客单价高、能讲故事"，但长期利润空间窄。

**判断 2：真正空白且 ⭐⭐⭐ 高合理性的方向有 5 个**

- a 第三方 LLM 可见度指数订阅
- b 垂直工具
- d2 GEO 战略咨询
- f 反 GEO 监测
- g 标准 / 协议推动方

这 5 个方向有共同特点：投入相对轻、客户付费意愿稳定、不直接面对效果承诺、扩张曲线非线性。

**判断 3：模式合理性高 ≠ 客户价值高**

e2 培训认证是典型的"客户价值低 + 模式合理性高"——焦虑变现是描述而非对错判断。这个市场会被填满，问题只是被谁填。如果你做这个，不要假装做的是别的事。

## 看探词科技在七层中的位置

探词手册里讲了四种产品形态：APIaaS / SaaS / RaaS / OEM。把它们映射到七层：

- APIaaS = a 层（监测）+ b 层（工具）边缘
- SaaS = b 层 + a 层
- RaaS = d1 效果代运营（主力）+ c 层（内容铺设是核心交付动作）
- OEM = e3 白标

它**没有**做的层：

- d2 战略咨询：探词不做"诊断后建议你不做 GEO"
- f 反 GEO 监测：探词手册有"舆情监控"产品计划，但还在持续优化方向，没真正落地
- g 标准 / 协议推动：完全没动

如果你是探词的同行或竞争对手，看到这张表第一反应应该是：**为什么大家都在挤 c/d，没人去 a / b / d2 / f / g？**

答案部分是：

- a / b 层需要工程能力，做服务出身的公司不擅长
- d2 战略咨询要求公司有真本事——能把 GEO 行业的局面看清楚，多数服务商自己都没看清
- f 反 GEO 监测的目标客户在公关 / 法务，跟做营销服务的销售路径完全不同
- g 标准 / 协议要影响力 + 长期愿景，多数中等规模公司不愿先动

这些"为什么"是空白存在的真实原因。但空白本身就是机会。

## 给读者的实操建议

**如果你是 GEO 服务商创业者**：

- 不要默认做 c/d 全栈，那是最拥挤的位置
- 看自己擅长什么：工程基因强 → a / b / 工具型；行业基因强 → d2 战略咨询 / 垂直陪跑；公关 / 法务背景 → f 反 GEO 监测；行业关系强 → g 协议推动

**如果你是 GEO 服务的买方**：

- 不要被"全栈服务"打动——多数所谓"全栈"是为了报价高，技术上反而难做精
- 区分卖方在哪一层：卖工具的（b 层）和卖代运营的（d 层）能力要求完全不同，不要用同一套标准评判
- 如果你的需求是"判断要不要做 GEO"，找的应该是 d2 类的咨询，不是 c/d 类的执行

**如果你是投资 GEO 行业的人**：

- c / d1 类公司估值天花板由人力成本决定，扩张是线性的
- a / b 类公司估值更接近 SaaS 倍数，但中文 GEO 场景里这两层尚未跑通
- f / g 类还在早期，但是结构性最有价值的方向
