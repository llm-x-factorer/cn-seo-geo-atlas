# 被严重低估的另一个生意：反 GEO 监测

**系列 5 / 6 · 2026-05-03**

GEO 行业里所有人都在做"帮品牌做 GEO"。这是进攻方。

但还有一个完全对称的生意没有人在做：**帮品牌识别"竞品在用 GEO 攻击我"、"AI 答案里出现关于我们的虚假信息"、"舆情通过 LLM 二次传播"**。这是防御方。

防御方的市场有多大？我的判断是——**与进攻方相当甚至更大**。

理由不是空想。下面从五个角度说清楚为什么这个生意被严重低估。

## 角度一：客户决策人完全不同

进攻型 GEO 的销售路径：

```
营销总监 / 市场增长部 → 看 ROI → 决策买单
```

反 GEO 监测的销售路径：

```
公关总监 / 法务总监 / 舆情监测部 → 评估风险 → 采购合规
```

**这两条路径不能复用**。具体差异：

- 决策周期：营销看季度，公关 / 法务看年度（合规预算编制流程）
- 决策依据：营销看 ROI，公关 / 法务看**风险案例 + 合规要求**
- 续费驱动：营销看续期 KPI，公关 / 法务看**合规延续 + 危机案例发生**
- 预算来源：营销预算来自市场费用，公关 / 法务预算来自风险管控费用——是两个独立的预算池

GEO 服务商如果想做"一站式覆盖进攻 + 防御"，多数会在这里翻车——销售团队不能同时讲两种语言。

## 角度二：客户付费心理完全不同

进攻方客户买的是**效果**："花了钱，我的品牌应该在 LLM 里被推荐"。但效果不可独立验证（客户没法证明"不花钱也可能这样"），所以续费时容易动摇。

防御方客户买的是**知情权 + 合规义务**：

- "我想知道 LLM 现在怎么描述我们"——这是知情权，监测报告本身就是产品交付
- "上市公司信息披露合规要求我们留痕"——这是合规义务，是必须的预算
- "竞品在通过 LLM 抹黑我"——这是危机响应，付费速度比常规营销快得多

**知情权 + 合规义务是稳定的购买动机**，不依赖效果验证。这绕开了进攻型 GEO 最致命的"效果不可独立验证"问题。

## 角度三：四类需求都是真需求

把"反 GEO 监测"按需求类型拆开，至少四类：

**N1 品牌可见度被动监测**：自家被 LLM 怎么描述，含正面 / 负面 / 错误信息。客户痛点是不知情焦虑 + 公关响应。

**N2 竞品攻击监测**：竞品是否在 LLM 答案中"踩"自家、做不公平比较、暗示自家产品有问题。客户痛点是品牌伤害 + 法律风险。

**N3 AI 幻觉监测**：LLM 编造的虚假信息——不存在的产品功能、错误的价格、假冒高管姓名、错误的法律事实、不实的医药效用。客户痛点是法律风险 + 监管合规。

**N4 AI 舆情扩散监测**：传统媒体的负面舆情通过 LLM 二次传播，扩大影响范围。客户痛点是危机扩散 + 二次发酵。

四类需求的客户重叠但付费心理不同。当前国内市场的"AI 舆情监测"产品多数只覆盖 N1，而且做得很浅——N2 / N3 / N4 几乎完全空白。

## 角度四：YMYL 是监管驱动的强需求

YMYL 是个搜索行业术语，全称 Your Money or Your Life，指**涉及用户金钱或生命健康的内容**。三大类：

- 医疗：用药、诊断、治疗方案
- 法律：法律咨询、合同条款、合规建议
- 金融：投资建议、财报数据、合规披露

YMYL 类的 AI 幻觉具有**直接合规风险**：

- LLM 编造的"X 药能治 Y 病"——可能引发医疗事故 + 药企责任
- LLM 给出错误法律建议——可能让律所或法律服务商被告
- LLM 编造的"X 公司财报"——可能引发证券误导
- LLM 答案与上市公司公开披露不一致——可能违反信息披露规则

中国《生成式人工智能服务管理办法》（国家网信办，2023 年 7 月落地）已经明确"生成式 AI 服务提供者要承担内容生产者责任"。后续配套细则会逐步压实"AI 内容真实性 / 来源可追溯"责任。

这意味着：YMYL 行业的反 GEO 监测**不依赖市场教育**——监管会推着客户来买。

医药公司、券商、律所、上市公司这些 YMYL 类持牌主体，已经有现成的合规预算。把"AI 监测"加入合规采购清单是顺势扩展，不是重新教育。

## 角度五：技术门槛低于进攻型 GEO

进攻型 GEO 需要七层全部能力：监测 + 工具 + 内容生产 + 媒体投放 + 战略咨询 + 客户成功 + 标准协议跟踪。

反 GEO 监测**只需要其中两层**：

- a 层（数据 / 监测）：跨 LLM 答案体抓取 + 语义识别 + 一致性指数计算 + 报告 + 告警系统
- f 层（防御工具）：幻觉检测器 / 跨 LLM 一致性指数器

**不需要**：

- c 层（内容生产 / 媒体投放）—— 防御型不做内容
- d 层（效果代运营）—— 防御型不承诺效果
- e 层（OEM 白标）—— 防御型客户不要工具私有化

技术上能力大部分已在传统舆情监测厂商技术栈中存在，需要补的只是 LLM 答案体抓取层（这本身是个工程问题，不是研究问题）。

**核心**：反 GEO 监测的产品形态是 SaaS 订阅 + 报告 + 告警，**毛利天花板由 SaaS 模型决定，不是人力天花板**。这与 d1 重人工代运营完全不同。

## 一个常见的反驳

有人会问：传统舆情监测厂商（蜂鸟、海纳、鹰眼、慧科 等）不就能延伸做这个吗？

不能直接延伸。具体技术差异：

| 维度 | 传统舆情监测 | 反 GEO 监测 |
|------|------------|-----------|
| 监测对象 | 公开网页 + 社交媒体 + 新闻 + 论坛 | LLM 答案体（动态生成） |
| 数据获取 | 爬虫 + RSS + API + UGC 抓取 | 模拟用户 query 拿答案体（每次都新生成） |
| 是否有 URL 锚点 | 有（每条数据可定位） | **无**（答案体不可定位 / 不可引用） |
| 复测性 | 同 URL 同时刻数据相同 | **低**（同 query 不同账号 / 时段答案不同） |
| 时间衰减 | 内容发布后稳定 | **每次新生成，无历史** |
| 个性化干扰 | 几乎无 | **强**（用户画像 / 历史会话影响答案） |

这些差异决定了**传统舆情厂商无法自然延伸**——他们的爬虫栈不能直接复用，监测方法学要重建。

这正是反 GEO 监测的技术壁垒所在。**真正动手做的人会形成不可被传统玩家轻易追上的领先**。

## 一个可能的风险：LLM 平台自建后台

参考 Google 给 SEO 行业做了 Search Console 后台。LLM 厂商可能上线"品牌方查询自家在 LLM 中表现"的官方后台。

如果这件事发生，**N1（品牌可见度）会被官方后台部分替代**——客户从豆包后台就能看到自己被怎么描述。

但 N2 / N3 / N4 仍然需要第三方：

- N2 竞品攻击：豆包不会主动告诉你"竞品的内容在攻击你"
- N3 AI 幻觉：豆包不会主动告诉你"我答案里关于你的内容是假的"
- N4 AI 舆情扩散：豆包不会主动告诉你"X 平台的负面舆情通过 LLM 扩散到了我"

这与 SEO 时代的剧本一致：Google Search Console 上线后，Ahrefs / Semrush / Moz 这些第三方 SEO 工具公司至今活跃。客户需要**跨平台对比** + **独立第三方报告**作为合规留痕。

## 给三类读者的实操建议

**如果你是大型企业的公关 / 法务总监**：

- 现在就在内部立项盘点："我们品牌在主流 LLM 答案里被怎么描述？" —— 即使不立刻采购，也要建立基线
- 把"AI 监测"加入年度合规预算编制
- 关注《生成式人工智能服务管理办法》的配套细则，特别是 YMYL 行业的执行细则

**如果你是 GEO 服务商创业者**：

- 反 GEO 监测是当前国内最被低估的方向之一
- 进入门槛比进攻型 GEO 低（不需要 c / d 层），且不背效果责任
- 销售路径与进攻型完全不同，团队和销售文化要重新搭建——不要试图在同一团队同时做两件事

**如果你是投资人**：

- 当前布局反 GEO 监测的公司极少，估值低
- 但客户群体明确（公关 / 法务 / 舆情监测部）、付费动机稳定（合规 + 知情权）、技术门槛低 + 不被平台广告位风险影响
- 这是 GEO 行业当前结构性最稳的位置
